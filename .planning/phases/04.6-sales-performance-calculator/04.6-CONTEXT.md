<domain>
## Phase Boundary

Ein interaktiver Sales Performance Calculator wird dem Dashboard hinzugefügt. Er zeigt dem User auf einen Blick was NERVE ihm finanziell bringt — echte Call-Daten aus seiner NERVE-History kombiniert mit einem Simulations-Modus mit Schiebereglern. Kein separater Tab, kein Page-Break: neuer Abschnitt im bestehenden Dashboard unterhalb der vorhandenen KPI-Karten.

**Was diese Phase liefert:**
- Neues DB-Feld: `ConversationLog.result` (gewonnen/verloren/offen) + `User.avg_deal_wert`
- `GET /api/performance` — liefert echte Metriken + Simulation-Defaults
- Dashboard-Sektion: KPI-Karten + 2 Charts (Balkendiagramm + Wachstumskurve) + Simulations-Schieberegler
- Deal-Wert-Onboarding: Modal beim ersten Dashboard-Besuch ohne avg_deal_wert

**Was diese Phase NICHT liefert:**
- Kein Umbau bestehender Dashboard-Elemente
- Kein Team-Dashboard / multi-user Aggregation
- Keine E-Mail-Reports
- Kein EWB-Tracking-Ausbau (Einwand-Erfolgsquote wird aus vorhandenen ConversationLog-Feldern abgeleitet)

</domain>

<decisions>
## Implementation Decisions

### Zwei Modi — exakt nach Briefing
- **Modus 1 (Echte Daten):** Standard-Ansicht. Daten aus ConversationLog (typ='live'). Calls/Woche = COUNT per Kalenderwoche. Closing-Rate = result='gewonnen' / (result='gewonnen' + result='verloren'). Einwand-Erfolgsquote = einwaende_behandelt / einwaende_gesamt. Deal-Wert = User.avg_deal_wert (User-Input).
- **Modus 2 (Simulation):** Schieberegler überschreiben alle Werte. Berechnung im Frontend (JS) — kein Server-Call beim Drehen. Ergebnis-Anzeige live aktualisieren.

### Neue Datenbank-Felder (Migration via _migrate())
- `conversation_logs.result` — VARCHAR(20), default=NULL, Werte: 'gewonnen' | 'verloren' | NULL (offen/Training)
- `users.avg_deal_wert` — INTEGER, default=NULL (Euros, kein Komma)

### API-Endpoint
- `GET /api/performance` — JSON-Response mit: `calls_pro_woche`, `closing_rate`, `einwand_erfolgsquote`, `avg_deal_wert`, `umsatz_pro_monat`, `forecast_3m`, `forecast_6m`, `forecast_12m`, `trend_prozent` (vs. Vormonat), `chart_data` (Balkendiagramm-Daten), `hat_daten` (Boolean — ob genug echte Daten vorhanden)
- Endpoint in bestehendem Blueprint: `routes/dashboard.py` oder neuer `routes/performance.py`

### Deal-Wert-Abfrage
- Wenn `User.avg_deal_wert` NULL: Modal beim ersten Aufruf des Dashboards
- Modal: Einzeiliges Input-Feld, Euros, Pflichtfeld, "Wie hoch ist dein durchschnittlicher Deal-Wert?"
- Nach Eingabe: PATCH /api/user/deal-wert speichern, Modal schließen, Calculator lädt mit Wert

### Call-Result-Erfassung (Gewonnen/Verloren)
- Im Live-Auswertungs-Flow (nach Session-Ende) Button-Gruppe ergänzen: "Gewonnen 🎉" | "Verloren" | "Offen lassen"
- Default: NULL (offen). User muss nicht taggen — Calculator funktioniert auch ohne (zeigt dann "Keine Closing-Rate verfügbar, nutze Simulation")
- Persistenz: PATCH /api/session/{id}/result

### Charts — Chart.js (bereits im Projekt via Training Analytics)
- **Balkendiagramm:** Bars pro Woche (letzte 8 Wochen), Y-Achse = Calls, optional zweite Reihe = Abschlüsse
- **Wachstumskurve:** Linie über 12 Monate Forecast, S-Kurve-Effekt durch leicht exponentielle Berechnung, Teal (#00D4AA) als Linienfarbe

### Simulation — 5 Schieberegler
| Schieberegler | Range | Default (aus echten Daten oder Fallback) |
|---|---|---|
| Calls pro Tag | 5–50 | Aus echten Daten oder 10 |
| Show-Rate | 10–100% | 70% |
| Setting → Closing Rate | 5–80% | Aus echten Daten oder 20% |
| Durchschnittlicher Deal-Wert (€) | 100–50.000 | User.avg_deal_wert |
| Arbeitstage pro Woche | 3–7 | 5 |

### Positionierung im Dashboard
- Unterhalb aller bestehenden Dashboard-Elemente (ROI-Tracker, KPI-Karten, Feed)
- Eigene Section mit Überschrift "Sales Performance" + Modus-Toggle (Echte Daten / Simulation)
- Kein eigener Tab, kein Redirect — scroll-into-view im gleichen Dashboard

### Claude's Discretion
- Genaue Formeln für S-Kurve / Forecast (z.B. linearer vs. leicht exponenteller Wachstumsfaktor)
- Mobile Responsive: Schieberegler stacken vertikal auf <768px
- Leere State wenn noch 0 Live-Calls: Placeholder mit "Starte deinen ersten Live-Call um echte Daten zu sehen" + direkt in Simulations-Modus wechseln
- Debounce auf Schieberegler (150ms) damit keine Performance-Probleme bei schnellem Drehen
- Rounding: Umsatz-Zahlen auf volle Euros, Closing-Rate auf 1 Dezimalstelle

</decisions>
