# Phase 04.8.1 Plan 05: Flask Integration + Frontend WS Client + Deployment Summary

**Flask issues Redis session tokens, frontend connects to Engine via native WebSocket with polling fallback, nginx routes /ws/ to Engine port 8001**

## Accomplishments

- Flask /live route generates single-use Redis session token (secrets.token_urlsafe, 60s TTL, HSET with user_id/mode/profile_id/language/profile_data)
- Frontend connectRTEngine() opens native WebSocket to /ws/{token}, falls back to HTTP polling after 3s timeout
- handleRTMessage() processes analysis_update, transcript, status, error — same UI update calls as pollErgebnis
- EWB clicks forwarded to Engine via sendRTControl for D-05 shadow ground truth logging
- beenden() sends stop_session control message and closes WebSocket before ending session
- systemd service runs Engine on 127.0.0.1:8001 with uvloop, Requires=redis.service
- nginx /ws/ location block before catch-all / routes WebSocket traffic to Engine

## Decisions Made

- Used sync redis-py in Flask (not async) — Flask runs threaded via gunicorn gthread, not asyncio
- Token generation wrapped in try/except with fallback to None — Redis unavailability does not break Flask
- window._pollingActive starts as false — polling only activates via startPolling() or 3s WS timeout safety net
- EWB clicks forwarded to Engine BEFORE the Flask API call — ensures ground truth reaches shadow logger even if Flask call is slow
- nginx /ws/ placed before / — nginx matches first matching location, so specific must come before catch-all

## User Setup Required

- **Redis:** Must be running on the VPS for session token handoff between Flask and Engine
  - Install: `sudo apt install redis-server && sudo systemctl enable redis-server`
  - Verify: `redis-cli ping` should return PONG
- **nerve-rt.service:** Copy to `/etc/systemd/system/nerve-rt.service`, then `sudo systemctl daemon-reload && sudo systemctl enable nerve-rt && sudo systemctl start nerve-rt`
- **nginx.conf:** Copy to `/etc/nginx/sites-available/nerve`, then `sudo nginx -t && sudo systemctl reload nginx`
