## Decisions Made
- TranscriptCallback is async Callable — streaming pattern requires async callbacks for non-blocking operation in asyncio event loop
- Connection handle is opaque `Any` type — adapter controls the concrete type, interface does not leak implementation details
- Speaker extraction uses majority-vote over word-level speaker counts — ported directly from existing deepgram_service.py `_get_speaker()` function
- `start()` and `finish()` wrapped with `asyncio.iscoroutine` check — Deepgram SDK version may return bool or coroutine (RESEARCH Pitfall 5)
