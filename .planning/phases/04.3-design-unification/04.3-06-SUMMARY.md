## What Was Built

### Task 1: Language buttons removed from Training page

The Training setup page previously showed language selection buttons (9 languages), which duplicated the language preference already available in Settings. These were removed:

- Deleted `.t-lang-btns`, `.t-lang-btn`, `.t-lang-btn.t-lang-active` CSS styles
- Deleted the `{% for code, lang in training_languages.items() %}` HTML language buttons block
- Deleted the `selectLanguage()` function (manipulated button active states)
- Deleted the `DOMContentLoaded` call to `selectLanguage()` for non-default languages
- Changed `currentLanguage` from a `let` variable to a `const` assigned directly from `{{ preferred_language | tojson }}`

The `TRAINING_LANGUAGES` constant was intentionally kept — it is still used by `startFreizeichen()` (ring tone hz/on/off values per locale) and `tUI` UI string lookups. All existing `currentLanguage` references continue to work correctly since the server-side value is the source of truth.

### Task 2: Sidebar Settings position and dark theme sweep

**Sidebar fix (DU-12):**
Added `<div style="flex:1"></div>` immediately before the Einstellungen nav item inside `.g-sidebar-inner`. This pushes Einstellungen to the bottom of the sidebar regardless of how many nav items appear above it. The spacer div is inside `.g-sidebar-inner` (not outside), so it does not break the collapsed sidebar CSS which hides `.g-sidebar-inner` content.

**Dark theme sweep (DU-03 completion):**
- `templates/app.html`: replaced all 52 occurrences of `#2dd4a8` with `#00D4AA` and all `rgba(45,212,168,` with `rgba(0,212,170,`
- `templates/dashboard.html`: replaced all 14 occurrences of `#2dd4a8` with `#00D4AA`
- AI gold `#d4a853` in app.html preserved unchanged

## Decisions Made

| Decision | Reason |
|----------|--------|
| Keep `TRAINING_LANGUAGES` constant | Still needed for freizeichen ring tones and tUI string lookups — not a language-switching UI element |
| `currentLanguage` as `const` from Jinja | All existing JS references work; variable is now read-only from server, eliminating stale client state |
| flex:1 spacer (not margin-top:auto) | Spacer must work inside flex column container; margin-top:auto on the link itself would also work but spacer div is more explicit and readable |
| Replace rgba(45,212,168 in app.html | Consistent with #00D4AA token change; rgba values with old channel values would produce wrong color |
