# Phase 4.3: Design Unification - Context

**Gathered:** 2026-04-04
**Status:** Ready for planning
**Source:** PRD Express Path (NERVE Design Unification Briefing)

<domain>
## Phase Boundary

Diese Phase entfernt den Light/Dark Mode komplett und vereinheitlicht das visuelle Erscheinungsbild der gesamten App auf ein einziges dunkles Theme. Außerdem wird ein kritischer Bug (Beenden-Button) gefixt und UI-Elemente werden aus globalen Bereichen (Header, Footer, Training) in die Einstellungen konsolidiert.

**Was diese Phase liefert:**
- Beenden-Button in Live-Assistent funktioniert
- Light Mode und Theme-Toggle komplett aus Codebase entfernt
- Alle 8 App-Bereiche auf einheitlichem Dark Theme
- Footer-Links, Header-Email/Logout, Training-Sprachauswahl in Einstellungen verschoben
- Settings-Button in Sidebar auf fixe Position gebracht

**Was NICHT in dieser Phase:**
- Keine neuen Features
- Keine Backend-Änderungen
- Kompaktmodus-Redesign (Nice-to-Have, niedrige Prio — nach allem anderen)
- Profil "Bearbeiten"-Button (Nice-to-Have)

</domain>

<decisions>
## Implementation Decisions

### KRITISCH — Beenden-Button Fix (DU-01)

- Beenden-Button im Live-Assistenten (`/live`) funktioniert nicht — User kann nicht zurück zum Dashboard
- Fix: Button muss zu `/dashboard` navigieren (oder `history.back()` falls das zuverlässiger ist)
- Priorität: Sofort fixen, vor allem anderen

### Light Mode entfernen (DU-02)

- **Komplett entfernen** — nicht verstecken, nicht auskommentieren — wirklich löschen
- Alle `prefers-color-scheme: light` Media Queries raus
- Theme-Toggle in `base.html` entfernen (und zugehöriges JS)
- CSS Custom Properties für Light Mode raus (alles was `--light-*` oder `@media (prefers-color-scheme: light)` ist)
- `nerve.css` ist das zentrale Stylesheet — dort liegt die gesamte CSS-Variable-Architektur
- Die App nutzt `.n-*` Komponenten-Klassen aus Phase 3.1

**Seite für Seite — konkrete Probleme:**

**Einstellungen (DU-04):** Dunkle Cards "schweben" auf hellem/grauem Hintergrund → Page-Background auf `#0D1117` oder `#161B22` setzen, alle 4 Tabs (Profil, Sicherheit, Benachrichtigungen, Billing) betroffen

**Analytics/Logs (DU-05):** Dunkle Tabelle auf hellem Hintergrund → gesamte Seite auf Dark Theme, Tabelle mit alternating rows (`#1C2333` / `#161B22`)

**Training (DU-06):** Dunkler Hintergrund gewährleisten, Anrufen-Button muss `#00D4AA` Teal auf dunklem BG sein — kontrastreich und deutlich sichtbar

**Profil-Editor (DU-07):** Dunkler Hintergrund, Input-Felder `background: #1C2333`, `border: 1px solid #2D3748`, `color: #FFFFFF`, Labels in `#FFFFFF` oder `#00D4AA` (nicht Orange)

**Hilfe-Center (DU-08):** Section-Headers in `#00D4AA` Teal (aktuell Orange), konsistent mit Dark Theme

### UI-Elemente umziehen (DU-09, DU-10, DU-11)

**Footer-Links entfernen (DU-09):**
- "Impressum · AGB · Datenschutz" aus Footer JEDER Seite entfernen
- Stattdessen: Neuer Tab oder Abschnitt "Rechtliches" in Einstellungen erstellen
- Einstellungen hat derzeit Tabs: Profil, Sicherheit, Benachrichtigungen, Billing — neuer Tab "Rechtliches" hinzufügen

**Header aufräumen (DU-10):**
- `admin@nerve.local` Email-Anzeige aus Header entfernen
- Logout-Button aus Header entfernen
- Beide in Einstellungen (Profil-Tab) verschieben — E-Mail-Anzeige im Profil-Tab, Logout-Button auch dort

**Sprachauswahl aus Training (DU-11):**
- Sprach-Buttons (Deutsch, English, etc.) von der Training-Seite entfernen
- In Einstellungen verschieben: Profil-Tab, neues Feld "Bevorzugte Sprache"
- Training nutzt dann die globale Spracheinstellung aus Einstellungen

### Settings-Button fixe Position (DU-12)

- Der Einstellungs-Button in der Sidebar springt zwischen Seiten
- Muss auf JEDER Seite an exakt der gleichen Pixel-Position sein
- Wahrscheinlich ein CSS-Problem (z.B. unterschiedliche Content-Höhen drücken den Button)
- Fix: `position: fixed` oder `margin-top: auto` in der Sidebar-Struktur

### Claude's Discretion

- Reihenfolge der Implementierung: Kritisch zuerst (Beenden-Button), dann CSS-Foundation (Light Mode raus + Dark Theme Variablen), dann Seite für Seite, dann UI-Elemente umziehen, dann Settings-Button
- Ob die Einstellungs-Tabs als neuer Tab "Rechtliches" oder als Abschnitt in einem bestehenden Tab implementiert werden — Claude entscheidet nach aktuellem Tab-Layout
- Ob `history.back()` oder `window.location.href = '/dashboard'` für den Beenden-Button geeigneter ist — Claude prüft den aktuellen Code

</decisions>
