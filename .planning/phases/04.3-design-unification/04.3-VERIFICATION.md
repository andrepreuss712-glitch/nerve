### Human Verification Required

The following behaviors require visual confirmation in a browser:

#### 1. Dashboard Back-Link Visible in Live Header

**Test:** Open `/live` while logged in.
**Expected:** A "← Dashboard" link is visible in the header, to the left of the mode badge. Clicking it navigates immediately to `/dashboard` without a confirm dialog.
**Why human:** Cannot verify visual placement programmatically.

#### 2. Sidebar Einstellungen Position Stable

**Test:** Navigate between `/dashboard`, `/training`, `/settings`, `/analytics`.
**Expected:** The Einstellungen nav item stays at the same vertical position (bottom of sidebar content area) on every page, regardless of how many nav items appear above.
**Why human:** Flex layout behavior requires visual inspection.

#### 3. Settings Rechtliches Tab Functional

**Test:** Open `/settings`, click the "Rechtliches" tab button.
**Expected:** Tab pane switches to show Impressum / AGB / Datenschutz links in teal. Clicking each navigates to the correct page.
**Why human:** Tab switching is JS-driven; link navigation requires browser.

#### 4. Training Language Uses Settings Preference

**Test:** Change language in Settings (Profil tab), start a training session.
**Expected:** Training scenarios and UI strings reflect the newly saved language preference without any language buttons visible on the Training page.
**Why human:** Requires live session interaction across two pages.
