## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual inline CSS, no framework |
| Preset | not applicable |
| Component library | none — custom components in Jinja2 + Vanilla JS |
| Icon library | Unicode emoji (inline in HTML) — no icon library |
| Font (headline) | `--font-headline`: Playfair Display, Georgia, serif — h1, h2, Alert-Titel, Dashboard-Begrüßung |
| Font (body) | `--font-body`: DM Sans, system-ui, sans-serif — Labels, Buttons, Body-Text, Navigation |
| Font (mono) | `--font-mono`: DM Mono, Courier New, monospace — Timer, Prozent-Zahlen, Metriken |
| Token delivery | **CSS Custom Properties only** — niemals Hex-Codes direkt in Templates |

Stack constraint: kein React, kein Framework-Wechsel. Alle neuen UI-Komponenten folgen dem bestehenden Flask + Vanilla JS + CSS-Custom-Properties-Pattern.

**Implementierungsregeln (aus `NERVE design.md` — verbindlich):**
1. Niemals Hex-Codes direkt in HTML-Templates schreiben
2. Niemals `style=""` Attribute für Farben nutzen
3. Immer CSS Custom Properties: `var(--color-amber)` statt `#E8922A`
4. Gradient als `var(--gradient-primary)` referenzieren, nicht inline
5. Alle Komponenten-Klassen in einer zentralen `design-tokens.css` definieren
6. Neue Farben erst in `NERVE design.md` definieren, dann verwenden — nie andersrum

## Spacing Scale

Declared values (multiples of 4, über CSS Custom Properties referenziert):

| Token | Value | CSS Var | Usage |
|-------|-------|---------|-------|
| xs | 4px | `var(--space-xs)` | Icon gaps, badge padding, tight inline spacing |
| sm | 8px | `var(--space-sm)` | Button padding (vertical), nav item gap, compact element spacing |
| md | 16px | `var(--space-md)` | Default form group margin, card internal padding, button horizontal padding |
| lg | 24px | `var(--space-lg)` | Page wrap horizontal padding, wizard nav-btns top padding, section bottom margin |
| xl | 32px | `var(--space-xl)` | Page wrap padding (`padding: 24px 32px`), layout horizontal breathing room |
| 2xl | 48px | — | Major section breaks, wizard bottom padding |
| 3xl | 64px | — | Page-level vertical spacing (full-screen views only) |

Border-Radius:

| Token | Value | CSS Var | Usage |
|-------|-------|---------|-------|
| sm | 4px | `var(--radius-sm)` | Badges, kleine Elemente |
| md | 6px | `var(--radius-md)` | Buttons — max für Buttons (keine Pillen) |
| lg | 12px | `var(--radius-lg)` | Karten, Panels |
| xl | 16px | `var(--radius-xl)` | Große Container |

Exceptions:
- Header height: 52px (existing fixed value — do not change)
- Sidebar width: 204px (existing fixed value — do not change)
- Wizard max-width: 580px (existing constraint — reuse for profile wizard)
- Touch target minimum: 44px height for primary action buttons
- Progress dots: 12px diameter (existing onboarding pattern — reuse in profile wizard)

## Typography

Größen-Hierarchie aus `NERVE design.md`:

| Role | Size | Font | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Hero / Panel-Headline | 2rem | `--font-headline` (Playfair Display) | bold | 1.2 | Live-Assist Panel-Headline, große Alert-Titel |
| Section Headline | 1.25rem | `--font-headline` (Playfair Display) | semibold (600) | 1.3 | h2, Seiten-Titel, Wizard-Step-h1 |
| Button / Label | 0.875rem | `--font-body` (DM Sans) | 500 (medium) | 1.0 | Buttons, Form-Labels, Nav-Items, Badges |
| Body | 0.875rem | `--font-body` (DM Sans) | 400 (regular) | 1.55 | Paragraph-Text, Tabellen-Zellen, Card-Body |
| Caption / Muted | 0.75rem | `--font-body` (DM Sans) | 400 (regular) | 1.5 | Sekundäre Metadaten, Timestamps, Hinweistexte |
| Timer / Zahlen | 1rem | `--font-mono` (DM Mono) | 400 | 1.0 | Timer, Prozent-Zahlen, Metriken |

**Wichtige Änderung gegenüber vorheriger Version:** Playfair Display ist **nicht mehr auf das Logo beschränkt** — es wird für alle h1, h2, Alert-Titel und die Dashboard-Begrüßung verwendet. DM Mono ist neu für numerische/metrische Darstellungen.

DM Sans weights in use: 400 (body) und 500 (buttons/labels). 700 bleibt für Ausnahmen (ALL-CAPS micro-labels mit letter-spacing). Kein 900.

Regel: Alle neuen UI-Komponenten verwenden `var(--font-body)` für Text und `var(--font-headline)` für Überschriften.

| Text muted | `var(--color-text-muted)` | `#555555` | Hints, deaktivierte States |
| Border | `var(--color-border)` | `rgba(255,255,255,0.10)` | Standard-Card-Border |
| Border gold | `var(--color-border-gold)` | `rgba(201,168,76,0.35)` | Akzent-Border, Beispiel-Boxen |
| Danger | `var(--color-danger)` | `#EF4444` | Fehler-States, Danger-Buttons |
| Success | `var(--color-success)` | `#22C55E` | Erfolgs-States |
| Live | `var(--color-live)` | `#E8922A` | Live-Indikator, aktive Session |

Gradient-Reservierung (`--gradient-primary`) für:
1. Primary CTA Button Background
2. Active navigation border (als `border-image`)
3. Progress indicator — aktiver Dot und abgeschlossene Linie
4. Selected card state (Tint via `rgba(232,146,42,.08)`)
5. Alert-Karte: Border-Left (3px, gradient via `border-image`)
6. Section-Label im Live/Active-State (Timer, Live-Dot)
7. Wizard step counter aktiver State
8. Beispiel-Boxen: Label-Text (`var(--color-amber)`) und Border-Tint (`var(--color-border-gold)`)
9. Badge-Live Hintergrund
10. Gauge-Arc Fill

Gradient ist **NICHT** für:
- Allgemeine Hyperlinks (use `color: inherit`)
- Inaktive/sekundäre Nav-Items
- Body-Text-Emphasis

### 1. Profile Wizard (`/profile/wizard` — neues Template `profile_wizard.html`)

Pattern basis: `templates/onboarding.html`

- 3-step wizard mit Progress-Dots (`.progress-dot` / `.progress-line` Pattern, 3 Dots)
- Wizard container: `max-width: 580px`, zentriert, `padding: var(--space-lg) var(--space-md) 48px`
- Step 1 — Branche & Rolle: `.branche-grid` (4-Spalten ≥420px, 2-Spalten darunter) + Role-Input
- Step 2 — Produkt & Firma: 3 Form-Inputs mit `.form-input`, generische Platzhalter (siehe Copywriting)
- Step 3 — Einwände: Checkliste 8–10 DACH B2B Einwände (`.level-card` Grid, Multi-Select) + freier `.form-textarea`
- Navigation: `.btn-back` / `.btn-next` Pattern aus onboarding, gestylt per `.btn-secondary` / `.btn-primary`
- Completion redirect: dashboard
- **Primary focal point:** Step-Heading (1.25rem / Playfair Display / semibold) zentriert über Progress-Dots

Card-Selektions-State (Branchen, Einwände):
```css
/* inactive */

border-radius: var(--radius-lg);

/* selected */

background: rgba(232, 146, 42, 0.08);
```
