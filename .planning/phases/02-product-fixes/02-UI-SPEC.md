## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, tight inline spacing |
| sm | 8px | Button padding (vertical), nav item gap, compact element spacing |
| md | 16px | Default form group margin, card internal padding columns, button horizontal padding |
| lg | 24px | Page wrap horizontal padding, wizard nav-btns top padding, section bottom margin |
| xl | 32px | Page wrap padding (`padding: 28px 32px`), layout horizontal breathing room |
| 2xl | 48px | Major section breaks, wizard bottom padding |
| 3xl | 64px | Page-level vertical spacing (full-screen views only) |

Exceptions:
- Header height: 52px (existing fixed value — do not change)
- Sidebar width: 204px (existing fixed value — do not change)
- Wizard max-width: 580px (existing constraint — reuse for profile wizard)
- Touch target minimum: 44px height for primary action buttons (mic-btn, t-call-btn pattern)
- Progress dots: 12px diameter (existing onboarding pattern — reuse in profile wizard)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px | 400 | 1.55 | Default paragraph text, table cells, card body copy, chat messages |
| Label | 11px | 700 | 1.0 | ALL-CAPS form labels, section headers, badge text (letter-spacing: .07em) |
| Heading | 22px | 700 | 1.2 | Page titles (`.page-title`), wizard step h1 at 26px on standalone pages |
| Display | 19px | 800 | 1.0 | Logo wordmark only — not for content headings |

Additional sizes in use (do not introduce new sizes beyond these):
- 10px / weight 700: badge labels, micro-labels (letter-spacing: .08em)
- 12px / weight 600: secondary metadata, timestamps, hint text
- 14px / weight 400: onboarding step body copy, card descriptions (line-height: 1.5)
- 15px / weight 800: primary CTA button text (training start button)

Rule: body text for all new UI components uses 13px / 400 / line-height 1.55. Headings inside cards use 14px / 700. Do not introduce sizes outside the existing set.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0c0c18` | Page background, card backgrounds, form inputs, overlays |
| Secondary (30%) | `#1a1a2e` | Card borders base, sidebar bg (`#0d0d14`), nav items bg, secondary buttons bg, bubble backgrounds |
| Accent (10%) | `#E8B040` | Reserved list below |
| Destructive | `#e05c5c` on `#1a0808` bg | Delete actions, error states, cancel/danger buttons only |

Accent `#E8B040` reserved for exactly these elements:
1. Active navigation item border and text
2. Primary CTA button background (text: `#0c0c18`)
3. Form input focus border
4. Progress indicator — active dot and completed line
5. Selected card state (`border-color: #E8B040`, `background: rgba(232,176,64,.08)`)
6. Section label text when it marks a live/active state (e.g. training timer, live dot)
7. Wizard step h1 logo sub-label ("SALES INTELLIGENCE")
8. Beispiel-Boxen: label text and border tint (`rgba(232,176,64,.33)`)
9. Profile wizard: wizard step counter active state

Accent is NOT used for:
- General hyperlinks (use `color: inherit`)
- Inactive/secondary nav items
- Body text emphasis

Border color for inactive containers: `#1a1a2e` (1px solid). Hover border upgrade: `#2e2e48`. Active/focus border: `#E8B040`.

Muted text color: `#6b6b80`. Subtler muted (hints): `#444466`. Body text: `#f0ebe0`.

## Component Inventory

New components required in this phase. All must follow existing class naming and inline CSS pattern.

### 1. Profile Wizard (`/profile/wizard` — new template `profile_wizard.html`)

Pattern basis: `templates/onboarding.html`

- 3-step wizard with progress dots (reuse `.progress-dot` / `.progress-line` pattern, 3 dots instead of 5)
- Wizard container: `max-width: 580px`, centered, `padding: 24px 16px 48px`
- Step 1 — Branche & Rolle: reuse `.branche-grid` (4-column at >=420px, 2-column below) + role text input
- Step 2 — Produkt & Firma: 3 form inputs with `.form-input` and generic placeholders (see Copywriting)
- Step 3 — Einwände: checklist of 8–10 predefined DACH B2B objections (`.level-card` grid pattern, multi-select allowed) + optional free-text `.form-textarea`
- Navigation: `.btn-back` / `.btn-next` pattern from onboarding
- Completion redirect: dashboard

### 2. Beispiel-Boxen (3 preview cards in onboarding step 1 area)

Pattern basis: `.example-box` from `onboarding.html`

Each card:
```
background: rgba(232,176,64,.05)
border: 1px solid rgba(232,176,64,.33)
border-radius: 8px
padding: 10px 14px
font-size: 12px
color: #8888a8
line-height: 1.65
```

Label row: `font-size: 10px; font-weight: 700; letter-spacing: .08em; text-transform: uppercase; color: #E8B040; margin-bottom: 4px`

Cards layout: stacked vertically (single column) on mobile, 3-column grid at >= 580px. Gap: 12px.

Content (static, no interaction):
- Card 1 label: "Gegenargument" — sample counter-text in body
- Card 2 label: "Kaufbereitschaft" — 0–100% gauge visual (CSS only, static at ~72%)
- Card 3 label: "Coaching-Tipp" — sample nudge text

### 3. Dashboard Style Selector (new card choice in onboarding)

Pattern basis: `.t-modus-card` / `.t-modus-active` from `training.html`, and `.cta-card` from onboarding

Two cards side by side (grid, gap 12px):
- "Vollständig" (pre-selected, `.selected` state on load) — label + short description
- "Fokus" — label + short description

Selected state: `border-color: #E8B040; background: rgba(232,176,64,.08)`

### 4. Training Mode Verification (existing UI — verify only, no new component)

Existing `.t-modus-card` / `.t-modus-active` pattern is correct. Verification task: ensure `selectModus('free')` hides `#t-helpBtn`. No visual changes needed unless a rendering bug is found.

### 5. Live-Mode Bug Fixes (existing UI — surgical fixes only)

No new component. Fix within existing `templates/app.html` and `static/app.js`:
- Script button: fix state logic, no visual redesign
- DSGVO banner: move timing, no visual redesign
- Compact mode circles: fix CSS rendering bug
- Toggle position: move element to correct position per existing intended layout
