### Task 2 — HTML Structure + JavaScript

**templates/training.html — 3 changes:**

**Change 1 — Grid wrapper:**
- `<div class="page-wrap">` content now wrapped in `<div class="t-training-layout">` with `.t-setup-wrap` (unchanged) as first column and `.t-analytics-panel` as second column.

**Change 2 — Analytics Panel HTML (7 cards):**
1. **Fortschritt**: 3 stat values (`t-stat-woche`, `t-stat-monat`, `t-stat-gesamt`) + avg duration label + streak badge.
2. **Wochenziel**: progress bar (`t-goal-bar`) + editable number input + Speichern button.
3. **Einwand-Heatmap**: `t-heatmap-grid` container (populated by JS).
4. **Verbesserungskurve**: `<canvas id="t-trend-chart">` + empty state text.
5. **KI-Empfehlung**: text element + "Jetzt trainieren" link button.
6. **Phrasen-Bank**: filter buttons container + phrases list + pagination (Zurueck/Weiter).
7. **Letzte Session**: `t-letzte-content` container (populated by JS).

**Change 3 — JavaScript IIFE (appended as separate `<script>` block):**
- Chart.js CDN: `https://cdn.jsdelivr.net/npm/chart.js@4/dist/chart.umd.min.js`
- `loadStats()`: fetches `/api/training/stats`, fills stat values, streak badge, wochenziel progress, triggers `renderHeatmap()` and `renderTrend()`.
- `renderHeatmap(heatmap)`: builds tile HTML with color class based on handle rate (≥70% gruen, ≥40% gelb, else rot, no data neutral). Click navigates to `/training?quick=1&einwand_typ=…`.
- `renderTrend(scores)`: creates/replaces Chart.js line chart (teal `#00D4AA`, fill, no axes). Shows empty state if <2 data points.
- `loadRecommendation()`: fetches `/api/training/recommendation`, fills text + shows button with `quick_url`.
- `loadPhrases(page, filter)`: fetches `/api/training/phrases` with page/filter params, renders phrase items, shows pagination if `pages > 1`.
- `renderPhraseFilters()`: renders 8 filter buttons (Alle + 7 Einwand-Typen), wires click to `loadPhrases(1, filter)`.
- `loadLastSession()`: fetches `/api/training/last-session`, renders session rows or empty state.
- Goal save: `t-goal-save` click POSTs to `/api/training/goal`, calls `loadStats()` on success.
- Phrases pagination: prev/next buttons wired to `loadPhrases()`.
- `checkQuickTraining()`: reads `?quick=1&einwand_typ=` URL params, pre-selects difficulty, scrolls to start button.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Removed unused `quote` variable in renderHeatmap**
- **Found during:** Task 2
- **Issue:** Plan's renderHeatmap JS declared `let quote = 0` in the outer scope but only the inner `if (gesamt > 0)` block set it, while `quoteText` was the value actually rendered. The variable was declared but its outer-scope value was never read — a silent lint issue.
- **Fix:** Removed the redundant outer `let quote = 0` declaration; only `quoteText` and `colorClass` are computed and used.
- **Files modified:** templates/training.html
- **Commit:** 1fe62f4
