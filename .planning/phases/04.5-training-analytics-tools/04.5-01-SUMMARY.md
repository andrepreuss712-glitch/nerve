# Phase 04.5 Plan 01: DB Schema + ConversationLog Persistence Summary

One-liner: Phrase model + weekly_goal column added; every training session end now persists a ConversationLog (typ='training'), saves wendepunkt_saetze as Phrase rows, and updates the user streak.

## What Was Built

### Task 1 — Schema Foundation

**database/models.py:**
- Added `Phrase` class with columns: `id`, `user_id` (FK → users), `session_id` (FK → conversation_logs, nullable), `text`, `objection_type`, `created_at`
- Added `User.weekly_goal = Column(Integer, default=5)` in Block 11

**app.py:**
- Added `('weekly_goal', 'INTEGER DEFAULT 5')` to `_migrate()` users block — existing DBs get the column on next startup

**routes/training.py:**
- Added `'profile_id': profile.id` to the `_sessions[g.user.id]` dict in `training_start()` so the FK is available at training end

### Task 2 — Persistence Logic

**services/training_service.py:**
- Extended `generate_scoring()` prompt to include `wendepunkt_saetze` field with instruction to extract 1-3 turning-point sentences
- JSON schema in prompt now includes the `wendepunkt_saetze` array with `text` and `einwand_typ` fields

**routes/training.py — training_end():**
- Added fallback `'wendepunkt_saetze': []` to error scoring dict
- Added ConversationLog persistence block BEFORE `_sessions.pop()`:
  - Creates `ConversationLog` with `typ='training'`, fills all required fields
  - `db_log.flush()` to get `log_entry.id` before Phrase inserts
  - Iterates `wendepunkt_saetze`, creates `Phrase` rows with FK to `log_entry.id`
  - Streak update: `days_diff==1` → increment, `days_diff>1` → reset to 1, `days_diff==0` → no change
  - Full try/except/finally with `db_log.close()` — error never breaks the training end flow
