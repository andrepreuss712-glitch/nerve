## What Was Built

### Task 1 ā€” Stats, Recommendation, Last-Session

**GET /api/training/stats:**
- Returns `sessions` object with `diese_woche`, `diesen_monat`, `gesamt` counts
- `avg_dauer_sekunden` computed from all training session durations
- `streak` and `streak_date` from `User.streak_count` / `User.streak_last_date`
- `wochenziel` from `User.weekly_goal` (default 5)
- `heatmap` dict keyed by all 7 EINWAND_TYPEN with `gesamt`/`behandelt` per type
- `trend_scores` array of last 10 `kb_end` scores in chronological order

**GET /api/training/recommendation:**
- Rule 1 (Schwaeche): worst objection type with < 40% handle rate and >= 3 attempts
- Rule 2 (Streak-Break): last training > 3 days ago
- Rule 3 (Trend): last 5 sessions avg > 15% better than previous 5
- Fallback: direct to hardest objection type
- No LLM call ā€” purely rule-based on ConversationLog data

**GET /api/training/last-session:**
- Returns most recent training ConversationLog with compact summary
- `haupt_einwand` extracted from first entry in `gegenargument_details` JSON
- `top_verbesserung` extracted from first item in `phasen_details.verbesserungen`
- Returns `session: null` if no training sessions exist

### Task 2 ā€” Phrases, Goal

**GET /api/training/phrases:**
- Paginated at 20 phrases per page, ordered by `created_at` desc
- Optional `?objection_type=` query param for equality filter (no LIKE/regex ā€” SQLAlchemy parameterized)
- Returns `total`, `page`, `pages` for frontend pagination controls
- Filtered by `user_id == g.user.id` ā€” no cross-user leakage (T-04.5-07)

**POST /api/training/goal:**
- Validates `goal` as integer, returns 400 with German error message if not parseable
- Validates range 1-30, returns 400 if out of range (T-04.5-06)
- Updates `User.weekly_goal` and commits
- Input validation happens before DB session is opened
