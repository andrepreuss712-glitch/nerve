  <files>templates/training.html, routes/training.py</files>
  <action>
  Der Quick-Training Flow muss den Einwand-Typ an den Server uebergeben, damit der KI-Gegner diesen Einwand priorisiert.

  **1. templates/training.html — Quick-Training Parameter beim Start uebergeben:**

  Finde die bestehende `startTraining()` Funktion (oder die Funktion die den POST /training/start ausfuehrt). Dort wird ein JSON-Body gebaut mit profile_id, schwierigkeit, modus, etc.

  Ergaenze diesen Body um ein optionales Feld:
  ```javascript
  // Am Anfang der startTraining() Funktion oder im globalen Scope:
  const _quickParams = new URLSearchParams(window.location.search);
  const _quickEinwand = _quickParams.get('einwand_typ');

  // Im POST body zu /training/start:
  body: JSON.stringify({
    ...existingFields,
    einwand_typ: _quickEinwand || null,  // Neues optionales Feld
  })
  ```

  Aktualisiere auch den `checkQuickTraining()` Code aus Plan 03: Wenn `quick=1` gesetzt ist, soll ein visueller Hinweis erscheinen (z.B. ein kleiner teal Banner ueber dem Start-Button):

  ```javascript
  if (params.get('quick') === '1' && einwandTyp) {
    // Show quick-training hint near start button
    const startBtn = document.getElementById('t-startBtn') || document.querySelector('.t-start-btn');
    if (startBtn) {
      const hint = document.createElement('div');
      hint.style.cssText = 'padding:8px 12px;background:rgba(0,212,170,0.1);border:1px solid #00D4AA;border-radius:8px;color:#00D4AA;font-size:13px;font-weight:600;margin-bottom:8px;text-align:center';
      hint.textContent = 'Quick-Training: ' + decodeURIComponent(einwandTyp);
      startBtn.parentNode.insertBefore(hint, startBtn);
      startBtn.scrollIntoView({behavior: 'smooth', block: 'center'});
    }
  }
  ```

  **2. routes/training.py — einwand_typ im Kunden-Prompt ergaenzen:**

  In `training_start()` (Zeile ~92), nach dem Lesen von `data`:
  ```python
  einwand_typ = data.get('einwand_typ')  # Optional: Quick-Training Einwand-Fokus
  ```

  Nach `customer_prompt = build_customer_prompt(...)` (ca. Zeile 124), wenn `einwand_typ` gesetzt ist, dem Prompt einen Hinweis anhaengen:
  ```python
  if einwand_typ:
      customer_prompt += f"\n\nWICHTIG: Fokussiere dich in diesem Training besonders auf den Einwand '{einwand_typ}'. Bringe diesen Einwand frueh und hartnäckig vor."
  ```

  Auch im Session-Dict (Zeile ~163) speichern:
  ```python
  'einwand_typ': einwand_typ,  # Quick-Training Fokus-Einwand
  ```

  Im JS, nachdem das Training gestartet wurde (nach erfolgreichem /training/start Response), die URL-Parameter bereinigen:

    window.history.replaceState({}, '', window.location.pathname);
  }
