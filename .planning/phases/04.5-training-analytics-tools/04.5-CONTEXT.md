### API-Endpoints (5 neue)
- `GET /api/training/stats` — Sessions (diese Woche/Monat/gesamt), Ø Dauer, Trainings-Frequenz, Heatmap-Daten, Streak (aus `User.streak_count`)
- `GET /api/training/recommendation` — Regelbasierte KI-Empfehlung (KEIN extra LLM-Call — berechnet aus stats: schlechtester Einwand-Typ, Streak-Break, Trenddaten)
- `GET /api/training/phrases` — Phrasen-Bank paginiert, filterbar via `?objection_type=`, `?page=`
- `POST /api/training/goal` — Wochenziel setzen `{goal: 5}` → `User.weekly_goal` updaten
- `GET /api/training/last-session` — Kompakte Zusammenfassung aus letztem `ConversationLog` mit `typ='training'`

### Phrasen-Extraktion (Post-Call Analyse)
- In `generate_scoring()` (training_service.py) Prompt um "Wendepunkt-Sätze"-Extraktion erweitern
- Response-JSON erhält neues Feld: `"wendepunkt_saetze": [{"text": "...", "einwand_typ": "..."}]`
- In `routes/training.py` nach `generate_scoring()` → Phrasen in `phrases` Tabelle speichern
- Haiku bleibt für Live-Loop; Sonnet-Call in `generate_scoring` trägt Phrasen-Extraktion kostenlos mit

### Charts
- Chart.js einbinden (CDN oder lokal) — nur für Sparkline/Balken-Charts in Fortschritts-Card und Verbesserungskurve
- Heatmap: reine CSS-Kacheln mit dynamischen Farben — KEIN Chart nötig
- Chart.js Version: 4.x (aktuell, lightweight)

### Quick-Training Flow
- Klick auf rote Heatmap-Kachel → `startTraining()` mit vorkonfiguriertem `einwand_typ`-Parameter
- "Schwachstelle trainieren"-Button → ruft `/api/training/recommendation` → startet mit schwächstem Einwand
- Technisch: URL-Parameter `?quick=1&einwand_typ=Zu+teuer` → training.html liest beim Start aus

### Wochenziel
- Reset jeden Montag (berechnet via `strftime('%W')` Vergleich, kein Cron nötig)
- Fortschritt = COUNT ConversationLog typ='training' in aktueller Kalenderwoche
- Streak-Logik: bestehende `User.streak_count` + `User.streak_last_date` → NICHT neu bauen

### Letzte Session Card
- Aus letztem `ConversationLog` mit `typ='training'`: Szenario-Name, Dauer, Gesamt-Score (aus `scoring` JSON falls gespeichert), Haupteinwand-Typ, Top-Feedback (1. `verbesserungen`-Eintrag aus Scoring)
- Link: `[Analyse ansehen →]` → navigiert zu `/dashboard#session-{log_id}` oder Dashboard-Logs

## Specific Ideas

### Einwand-Typen (aus bestehender Konfiguration)
Die 7 Einwand-Typen aus dem Live-Assistenten und Training:
`Keine Zeit`, `Zu teuer`, `Kein Interesse`, `Haben schon`, `Muss überlegen`, `Schicken Sie Unterlagen`, `Kein Bedarf`

### Heatmap-Farb-Logik
- Erfolgsquote > 70% → grün (`#00D4AA`)
- Erfolgsquote 40-70% → gelb/orange (`#F59E0B`)
- Erfolgsquote < 40% → rot (`#F85149`)
- Keine Daten → neutral (`#E5E7EB` mit grauem Text)

### KI-Empfehlung — Regelbasierte Logik
1. Schlechtester Einwand-Typ (< 40% + >= 3 Versuche) → "Du hast [Einwand] {N}x verloren — üb das jetzt"
2. Streak-Break (streak_last_date > 3 Tage her) → "Seit {N} Tagen nicht trainiert — 10 Minuten reichen"
3. Positiver Trend (letzte 7 Tage vs. vorherige 7 Tage > +15%) → "Deine [Einwand]-Rate ist von X% auf Y% gestiegen"
4. Fallback (keine klare Schwäche) → "Trainiere heute den härtesten Einwand: [schlechtester Typ]"

### Gegenargument_details JSON-Format (bestehend)
```json
[{"einwand_typ": "Zu teuer", "behandelt": true, "gegenargument": "..."}, ...]
```
`behandelt: true` = Einwand erfolgreich behandelt
