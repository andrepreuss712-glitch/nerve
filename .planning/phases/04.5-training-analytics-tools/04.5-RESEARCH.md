## Summary

Phase 4.5 baut auf einer kritischen Vorarbeit auf: Der `/training/end`-Endpoint speichert heute nur Punkte/Level. Alle Analytics-Features hängen davon ab, dass jede beendete Training-Session einen `ConversationLog`-Eintrag mit `typ='training'` erzeugt. Wave 0 der Implementierung muss daher ausschließlich diese Persistenz liefern, bevor irgendein Analytics-Endpoint funktionieren kann.

Das bestehende Schema hat fast alles was gebraucht wird: `ConversationLog` besitzt `dauer_sekunden`, `typ`, `gegenargument_details` (JSON), `einwaende_behandelt`, `einwaende_gesamt`. `User` hat bereits `streak_count` und `streak_last_date`. Zwei Additions sind nötig: die `phrases` Tabelle (neue Tabelle über `init_db`) und `users.weekly_goal` (Migration via `_migrate()`-Muster).

Das Layout-Eingriff ist chirurgisch: `#t-setup` enthält aktuell nur `.page-wrap > .page-header + .t-setup-wrap`. Die bestehende `.t-setup-wrap` (max-width: 580px) wird in ein Zwei-Spalten-Grid eingebettet. Phase 2/3/4 (`#t-einwahl`, `#t-chat`, `#t-scoring`) sind `.t-overlay`-Divs mit `position:fixed` — sie werden durch das Grid nicht beeinflusst.

**Primary recommendation:** Wave 0 = ConversationLog-Persistenz + DB-Schema. Wave 1 = 5 API-Endpoints + generate_scoring()-Erweiterung. Wave 2 = HTML-Panel + Chart.js + JS. Wave 3 = Quick-Training URL-Parameter.


Das bestehende Format für Live-Sessions:

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Wochenberechnung | Custom Datum-Arithmetik | `strftime('%W')` + `datetime.timedelta` | Eingebaut, DST-safe |
| Chart-Rendering | SVG-Pfad-Berechnung | Chart.js 4.x (CDN) | CONTEXT.md-Entscheidung |
| JSON-Parsing für gegenargument_details | Custom Parser | `json.loads()` + try-except | Bestehendes Muster im Codebase |
| DB-Migration | DROP/CREATE TABLE | `_migrate()` Muster mit ALTER TABLE + silent-except | Bestehender Ansatz, SQLite-kompatibel |

## Common Pitfalls

### Pitfall 1: Session-Dict löschen vor ConversationLog-Save
**Was falsch geht:** `_sessions.pop(g.user.id, None)` passiert in `training_end()` bei Zeile 367, die Punkte-Speicherung ist danach (Zeile 370+). ConversationLog-Speicherung wird vermutlich auch nach dem Pop eingefügt — die Session-Daten sind dann weg.
**Root Cause:** `session`-Variable ist eine lokale Referenz die vor `pop()` gesetzt wird (Zeile 312) — das ist safe. `session` bleibt valid nach `pop()`. Kein echtes Problem, aber Reihenfolge im Code muss klar sein.
**Vermeidung:** ConversationLog erstellen aus der lokalen `session`-Variable (die vor pop() geholt wurde), dann `pop()`, dann Punkte-Update. Alle drei im gleichen Try-Block.

### Pitfall 2: profile_id fehlt im Session-Dict
**Was falsch geht:** `_sessions[g.user.id]` in `training_start()` (Zeile 163) speichert `profile_name` aber NICHT `profile_id`.
**Root Cause:** `profile_id` wurde nicht in das Dict aufgenommen.
**Vermeidung:** In `training_start()` `'profile_id': profile.id` zum Dict hinzufügen (minimale Änderung). Dann in `training_end()` verfügbar.

### Pitfall 3: generate_scoring() gibt kein wendepunkt_saetze zurück bei Fehler
**Was falsch geht:** Bei Claude-Timeout oder JSON-Parse-Fehler fällt `training_end()` auf den Fallback zurück (Zeile 331-335) — kein `wendepunkt_saetze` im Fallback-Dict.
**Root Cause:** Fallback-Dict ist hardcoded ohne das neue Feld.
**Vermeidung:** Fallback-Dict immer mit `'wendepunkt_saetze': []` ergänzen.

### Pitfall 4: Chart.js wird in Overlays initialisiert die display:none sind
**Was falsch geht:** Chart.js auf einem `<canvas>` initialisieren das nicht sichtbar ist (display:none Eltern) → Canvas-Größe = 0 → Chart rendert nicht.
**Root Cause:** Phase 1 (Setup) ist sichtbar, aber die Analytics-Cards darin laden erst Daten nach Page-Load via Fetch. Kein Problem wenn Canvas schon im DOM ist.
**Vermeidung:** Chart erst initialisieren wenn Stats-API-Response zurückkommt. `if (!chartInstance) { /* init */ }` Guard.

### Pitfall 5: Heatmap mit nicht-existierenden Einwand-Typen
**Was falsch geht:** Heatmap rendert für alle 7 Einwand-Typen, aber User hat vielleicht nur für 2 Daten.
**Root Cause:** Keine Daten = Division durch Null bei Erfolgsquote.
**Vermeidung:** Default `{'behandelt': 0, 'gesamt': 0}` pro Typ. Wenn `gesamt == 0` → neutrale Farbe `#E5E7EB`.

### Pitfall 6: SQLite-Tabellen-Erstellung Race Condition
**Was falsch geht:** `phrases`-Tabelle wird via `Base.metadata.create_all()` erstellt — aber nur wenn das Modell beim Import bekannt ist.
**Root Cause:** `app.py` importiert models indirekt. Wenn `Phrase`-Klasse nicht in `database/models.py` importiert ist wenn `init_db()` läuft, wird Tabelle nicht erstellt.
**Vermeidung:** `Phrase`-Modell in `database/models.py` definieren (nicht in anderem File). `from database.models import Phrase` am Anfang von `routes/training.py` ergänzen.

### ConversationLog in training_end() speichern

```python
# Source: direkt aus routes/training.py Struktur abgeleitet [VERIFIED]
# Nach scoring = generate_scoring(...) und VOR _sessions.pop():

wendepunkt_saetze = scoring.get('wendepunkt_saetze', [])
ga_details = json.dumps([
    {'einwand_typ': ws['einwand_typ'], 'behandelt': True, 'gegenargument': ws['text']}
    for ws in wendepunkt_saetze
    if ws.get('einwand_typ') and ws.get('text')
])

try:
    db_log = get_session()
    from database.models import ConversationLog as CLog, Phrase as PhraseModel
    log = CLog(
        user_id        = g.user.id,
        org_id         = g.org.id,
        profile_id     = session.get('profile_id'),   # nur wenn training_start() ergänzt wurde
        profile_name   = session['profile_name'],
        started_at     = session['started_at'],
        ended_at       = datetime.now(),
        dauer_sekunden = int((datetime.now() - session['started_at']).total_seconds()),
        hilfe_genutzt  = hilfe_count,
        kb_end         = scoring.get('gesamt_score', 0),
        einwaende_gesamt    = len(wendepunkt_saetze),
        einwaende_behandelt = len(wendepunkt_saetze),
        gegenargument_details = ga_details,
        typ            = 'training',
        session_mode   = session.get('modus', 'guided'),
    )
    db_log.add(log)
    db_log.flush()   # log.id verfügbar für Phrase FKs

    for ws in wendepunkt_saetze:
        if ws.get('text') and ws.get('einwand_typ'):
            p = PhraseModel(
                user_id        = g.user.id,
                session_id     = log.id,
                text           = ws['text'],
                objection_type = ws['einwand_typ'],
            )
            db_log.add(p)
    db_log.commit()
except Exception as ex:
    print(f"[Training] ConversationLog-Speicherung Fehler: {ex}")
finally:
    db_log.close()
```

```python
@training_bp.route('/api/training/stats')
@login_required
def api_training_stats():
    from datetime import timedelta, date
    db = get_session()
    try:
        user = db.get(User, g.user.id)
        heute = datetime.now()
        woche_start = heute - timedelta(days=heute.weekday())
        monat_start = heute.replace(day=1, hour=0, minute=0, second=0)

        basis = db.query(ConversationLog).filter(
            ConversationLog.user_id == g.user.id,
            ConversationLog.typ == 'training',
        )
        sessions_gesamt       = basis.count()
        sessions_diese_woche  = basis.filter(ConversationLog.created_at >= woche_start).count()
        sessions_diesen_monat = basis.filter(ConversationLog.created_at >= monat_start).count()

        alle_logs = basis.order_by(ConversationLog.created_at.desc()).all()
        avg_dauer = 0
        if alle_logs:
            dauern = [l.dauer_sekunden or 0 for l in alle_logs if l.dauer_sekunden]
            avg_dauer = int(sum(dauern) / len(dauern)) if dauern else 0

        # Heatmap: Erfolgsquote pro Einwand-Typ
        einwand_stats = {}
        EINWAND_TYPEN = ['Keine Zeit', 'Zu teuer', 'Kein Interesse',
                         'Haben schon', 'Muss überlegen',
                         'Schicken Sie Unterlagen', 'Kein Bedarf']
        for et in EINWAND_TYPEN:
            einwand_stats[et] = {'gesamt': 0, 'behandelt': 0}
        for log in alle_logs:
            if not log.gegenargument_details:
                continue
            try:
                for ga in json.loads(log.gegenargument_details):
                    et = ga.get('einwand_typ', '')
                    if et in einwand_stats:
                        einwand_stats[et]['gesamt'] += 1
                        if ga.get('behandelt'):
                            einwand_stats[et]['behandelt'] += 1
            except Exception:
                pass

        # Trend: letzte 10 Sessions gesamt_score (kb_end)
        letzte_scores = [l.kb_end or 0 for l in reversed(alle_logs[:10])]

        return jsonify({
            'ok': True,
            'sessions': {
                'diese_woche':  sessions_diese_woche,
                'diesen_monat': sessions_diesen_monat,
                'gesamt':       sessions_gesamt,
            },
            'avg_dauer_sekunden': avg_dauer,
            'streak':       user.streak_count or 0,
            'streak_date':  user.streak_last_date.isoformat() if user.streak_last_date else None,
            'wochenziel':   user.weekly_goal or 5,
            'heatmap':      einwand_stats,
            'trend_scores': letzte_scores,
```

```python
                    if not et:
                        continue
                    if et not in einwand_stats:
                        einwand_stats[et] = {'gesamt': 0, 'behandelt': 0}
                    einwand_stats[et]['gesamt'] += 1
                    if ga.get('behandelt'):
```
