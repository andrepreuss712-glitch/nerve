# Phase 04.9 Plan 04: Frontend Training UI Upgrade — Personality Cards + Mood + Hangup UX Summary

Complete training frontend upgrade: personality type card selection with random generation and briefing display, scenario step in setup flow, mood indicator bar for Einsteiger mode, red-flash hangup overlay with auto-transition to scoring, letzte-chance warnings, Experte personality reveal in scoring, and post-training save prompt for generated personalities.

## What Was Built

### Task 1: Setup flow — personality type cards + scenario step + briefing

**templates/training.html (HTML):**
- Added `<div id="t-typ-section">` with a CSS grid card container `id="t-typ-cards"`, random generate button `id="t-typ-random-btn"`, and briefing card `id="t-typ-briefing"` — inserted between Modus and Schwierigkeit sections per D-12 order
- Added `<div id="t-szenario-section">` with `id="t-szenario-list"` grid and "Ohne Szenario" skip button — inserted between Schwierigkeit and Modus sections (both new sections hidden by default via `style="display:none"`)
- Added `id="t-diff-section"` to diff grid wrapper for smooth-scroll target

**templates/training.html (JavaScript):**
- Added state vars: `selectedPersonalityTypeId`, `generatedPersonality`, `selectedScenarioId`
- `loadPersonalityTypes()` — fetches `/api/training/personalities`, renders cards with icon/name/kurzbeschreibung/custom badge
- `selectTyp(id, el)` — sets selectedPersonalityTypeId, highlights card, hides briefing
- `generateRandomPersonality()` — POSTs to `/api/training/personalities/generate`, shows briefing card with returned `briefing` text, clears card selection
- `confirmGeneratedPersonality()` — smooth-scrolls to difficulty section
- `loadSzenarienForTypFlow()` — fetches `/training/scenarios`, renders cards with name/beschreibung/NERVE Standard badge
- `selectSzenario(id, el)` — sets selectedScenarioId, highlights card
- `skipSzenario()` — clears selectedScenarioId and card highlights
- `selectModus()` updated: calls `loadPersonalityTypes()` and shows `t-typ-section`
- `selectDiff()` updated: calls `loadSzenarienForTypFlow()` and shows `t-szenario-section`
- `startTraining()` updated: `startPayload` includes `personality_type_id` (if set), `generated_personality` (if generated without saved ID), `scenario_id` (selectedScenarioId || selectedScenario)

### Task 2: Chat phase — mood indicator + hangup UX + letzte-chance + save prompt

**templates/training.html (CSS):**
- Added `@keyframes hangupFlash` — 0% opacity:0, 20-80% opacity:1, 100% opacity:0 (red flash animation)

**templates/training.html (HTML):**
- Added `id="t-mood-indicator"` in chat header (right side, flex row): Stimmung label + `id="t-mood-bar"` (80px bar) + `id="t-mood-fill"` (animated fill) + `id="t-mood-value"` (text) — hidden by default
- Added `id="t-hangup-overlay"` fixed full-screen div with rgba(220,38,38,0.3) background and z-index:1000, pointer-events:none — hidden by default

**templates/training.html (JavaScript):**
- `updateMoodIndicator(stimmung)` — maps -5..+5 to 0-100% width; colors: red (≤-3), orange (-2..0), yellow (1..3), green (4..5); shows/hides indicator div
- `sendToKI()` updated:
  - After response: if `data.schwierigkeit === 'leicht'` and stimmung present → calls `updateMoodIndicator()`
  - If `data.letzte_chance`: adds orange border-left and warning text to last `.t-msg-in`
  - If `data.aufgelegt`: shows overlay with hangupFlash animation, disables all inputs, calls `confirmEndTraining()` after 2000ms
- `initChat()` updated: shows mood indicator on training start when `selectedDiff === 'leicht'` and `data.stimmung` present (D-07)
- `confirmEndTraining()` updated after `buildScoring(data)`:
  - If `data.personality_revealed`: inserts reveal card at top of `t-scoringInner` with icon/name/kurzbeschreibung (D-05)
  - If `generatedPersonality && !selectedPersonalityTypeId`: appends save prompt div with name, optional comment input, Speichern/Verwerfen buttons (D-03)
- `saveGeneratedPersonality()` — POSTs to `/api/training/personalities/save` with name/icon/kurzbeschreibung/attribute/kommentar; on success replaces save div with "✓ Gespeichert!"
- `resetTraining()` updated: clears `selectedPersonalityTypeId`, `generatedPersonality`, `selectedScenarioId`; resets mood indicator and hangup overlay DOM state

## Deviations from Plan

**1. [Rule 2 - Enhancement] Both tasks committed together**
- Tasks 1 and 2 both modify training.html exclusively. Combined into single commit to avoid a partial state where mood indicator code references functions not yet present (selectDiff shows scenario section, buildScoring checks personality_revealed).

**2. [Rule 1 - Bug] `selectModus()` shows t-typ-section without loadPersonalityTypes() guard**
- The plan called loadPersonalityTypes() when modus is selected. Implemented as specified — cards rendered async into hidden section, ready when user scrolls.

**3. [Rule 2 - Missing] Mood indicator missing from `initChat()`**
- Plan only showed mood in `sendToKI()` respond handler. Added initial mood display in `initChat()` using `data.stimmung` from training start response for Einsteiger — without this, the bar stays hidden until the first response even if the session starts with a non-neutral mood.

**4. [Rule 2 - Missing] `resetTraining()` DOM cleanup**
- Plan did not mention resetting overlay/indicator on reset. Added explicit DOM cleanup to prevent stale UI state on repeated training sessions.

## Known Stubs

None — all UI elements are fully wired:
- `t-typ-cards` populated via `loadPersonalityTypes()` fetch
- `t-szenario-list` populated via `loadSzenarienForTypFlow()` fetch
- `t-mood-fill`/`t-mood-value` updated via `updateMoodIndicator()` from live respond data
- `personality_revealed` and save prompt rendered from real `/training/end` response data

## Threat Surface Scan

No new network endpoints introduced. This plan is pure frontend — all fetch calls go to endpoints established in Plan 03:
- T-04.9-10 (Info Disclosure/hidden): Frontend receives only 'Unbekannt' for hidden personalities until `/training/end` — reveal only shown from end response, not during chat. Fully mitigated client-side.
- T-04.9-11 (Tampering/save input): `saveGeneratedPersonality()` sends name/icon/kurzbeschreibung/attribute — server-side truncation applied per Plan 03 T-04.9-06. Client-side kommentar is freetext with no execution context.
