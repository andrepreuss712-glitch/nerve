# Phase 04.9 Plan 02: Personality-Driven AI Core — build_personality_prompt() + generate_response_with_mood() Summary

Personality-driven Claude prompt builder with difficulty-aware auflege rules and structured JSON response function with mood tracking, hangup signals, and graceful fallback — the core AI logic for Phase 04.9 training mode.

## What Was Built

### Task 1: build_personality_prompt() + AUFLEGE_REGELN + PERSONALITY_MOOD_PROMPT_SUFFIX

**services/training_service.py:**
- Added `AUFLEGE_REGELN` dict with 3 difficulty entries:
  - `leicht`: letzte_chance warning at -4, aufgelegt only at -5 after warning
  - `mittel`: letzte_chance on soft triggers at <= -3, aufgelegt on hard triggers without warning or at -5
  - `schwer`: no letzte_chance ever, aufgelegt at -4 (stricter threshold)
- Added `PERSONALITY_MOOD_PROMPT_SUFFIX` f-string template with:
  - Stimmungs-Tracking section (11 mood change rules, +/- deltas)
  - Auflege-Logik section (difficulty-specific rules via placeholder)
  - Hard trigger section (personality-specific via placeholder)
  - JSON output format specification
- Added `build_personality_prompt(profile_data, personality_data, schwierigkeit, current_mood, sprache, szenario)`:
  - Builds complete system prompt from PersonalityType.attribute JSON data
  - Includes personality name, verhaltensregeln, position, vorgeschichte, beispiel_reaktionen
  - Includes profile context (produkt, branche, einwaende)
  - Appends optional szenario context block
  - Appends PERSONALITY_MOOD_PROMPT_SUFFIX with all placeholders filled

### Task 2: generate_response_with_mood()

**services/training_service.py:**
- Added `generate_response_with_mood(conversation_history, system_prompt, current_mood, schwierigkeit)`:
  - Builds messages list from conversation history (same speaker mapping as generate_response)
  - Calls claude-haiku-4-5-20251001 with max_tokens=300
  - Extracts JSON via `find('{')` / `rfind('}')` pattern (consistent with generate_scoring)
  - Returns dict: `{text, neue_stimmung, aufgelegt, letzte_chance}`
  - Clamps neue_stimmung to [-5, +5]
  - Casts aufgelegt/letzte_chance to bool (T-04.9-03 mitigation)
  - Two fallback paths: no-braces and JSONDecodeError — both return plain text with unchanged mood
  - Logs parse failures with `[Training]` prefix per codebase convention
