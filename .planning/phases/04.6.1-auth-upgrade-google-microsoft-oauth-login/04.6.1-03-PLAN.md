<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Landing-Page | Untrusted Query-Params (oauth_error) werden in DOM gerendert |
| Landing-Page → /auth/google[/microsoft] | Top-Level-Navigation, kein fetch — Browser folgt OAuth-Flow nativ |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04.6.1-16 | Tampering | oauth_error Query-Param Rendering | mitigate | Errortext kommt aus client-seitiger Lookup-Map (errMap), NICHT direkt aus URL → kein XSS. Wert wird per `textContent` (nicht innerHTML) gesetzt → kein HTML-Injection. |
| T-04.6.1-17 | Information Disclosure | OAuth-Buttons als <a> mit href | accept | href ist statisch `/auth/google` bzw. `/auth/microsoft` — kein User-Input, kein Open-Redirect-Vector |
| T-04.6.1-18 | Spoofing | Externe SVG-Logos | mitigate | Inline-SVG, KEIN externer CDN-Load — kein Tracking, keine 3rd-Party-Abhängigkeit für die Anzeige |
| T-04.6.1-19 | Repudiation | Klick auf OAuth-Button | accept | Standard-Browser-Navigation, kein eigenes Logging nötig — Server-Side `[OAuth] ... login` print reicht für Audit |
</threat_model>
