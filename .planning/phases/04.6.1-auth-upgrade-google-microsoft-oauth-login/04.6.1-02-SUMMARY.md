# Phase 04.6.1 Plan 02: OAuth-Flow Google + Microsoft via authlib Summary

**One-liner:** Server-seitiger OIDC-Login für Google + Microsoft via authlib mit OIDC-Discovery, ProxyFix für https-Redirect-URIs hinter nginx, und shared `_oauth_login_or_create` Helper der bestehende Email-User per Email-Match verknüpft oder Neuanlagen via `_create_org_and_user` durchführt.

## What Was Built

### Task 1: Dependencies + Config + Env
- `requirements.txt`: `authlib>=1.3.0` angehängt; `pip install` zog authlib 1.6.9 + cryptography ohne Fehler.
- `config.py`: 4 OAuth-Vars (`GOOGLE_CLIENT_ID/SECRET`, `MICROSOFT_CLIENT_ID/SECRET`) aus `os.environ`, leerer String als Default. Keine Logs der Werte.
- `.env.example`: 4 neue Vars als Platzhalter dokumentiert. Echte Secrets aus CONTEXT.md NICHT committed.
- **Commit:** `672049a`

### Task 2: routes/oauth.py
- Modul-globale `oauth = OAuth()` Instanz, Blueprint `oauth_bp`.
- `init_oauth(app)` registriert Google + Microsoft via `server_metadata_url` (OIDC Discovery → JWKS + Signatur-Validierung automatisch). Logs `[OAuth] {provider} client registered` oder `… missing — disabled` bei fehlenden Credentials.
- 4 Routes: `/auth/google`, `/auth/google/callback`, `/auth/microsoft`, `/auth/microsoft/callback`.
- `_oauth_login_or_create(*, provider, oauth_id, email, vorname, nachname, avatar_url)` — provider-agnostischer Helper:
  - Email leer → `_oauth_error_redirect('no_email', ...)`
  - Email-Match in DB → inactive-Check → OAuth-Felder idempotent nachziehen → `session.clear()` → `_login_user` → Redirect `/onboarding` (wenn `onboarding_done=False`) sonst `/dashboard`
  - Kein Match → `_create_org_and_user` mit `passwort_hash=''` Sentinel + OAuth-Feldern + Platzhalter-Firmenname `<Name> (Workspace)` → `session.clear()` → `_login_user` → Redirect `/onboarding`
- Google-Callback: `email_verified` MUSS True sein (T-04.6.1-12).
- Microsoft-Callback: KEIN email_verified-Check; `name` wird pragmatisch in vorname/nachname gesplittet; `oid` als Fallback für `sub`.
- Try/except um `authorize_access_token` → Redirect `/?modal=login&oauth_error=<code>` mit deutscher flash-Message.
- **Commit:** `e1dbb90`

### Task 3: app.py — Registration + ProxyFix
- ProxyFix war NICHT aktiv → eingefügt direkt nach `app = Flask(__name__)`:
  ```python
  from werkzeug.middleware.proxy_fix import ProxyFix
  app.wsgi_app = ProxyFix(app.wsgi_app, x_proto=1, x_host=1, x_for=1)
  ```
- Import `from routes.oauth import oauth_bp, init_oauth` nach `performance_bp`.
- `app.register_blueprint(oauth_bp)` + `init_oauth(app)` nach `performance_bp` Registrierung.
- App-Start-Log verifiziert:
  - `[OAuth] Google credentials missing — /auth/google disabled` (lokal ohne .env)
  - `[OAuth] Microsoft credentials missing — /auth/microsoft disabled`
  - `oauth.google_login` ist im `app.url_map` enthalten
  - `app.wsgi_app` ist `_SocketIOMiddleware` (das ProxyFix-Wrapper enthält — ProxyFix wird vor SocketIO-Init zugewiesen, SocketIO wickelt es korrekt)
- **Commit:** `aa62f78`

## Threat Mitigations Applied

| Threat ID | Mitigation |
|-----------|------------|
| T-04.6.1-06 (Spoofing CSRF) | `authlib.authorize_access_token()` validiert state + nonce automatisch |
| T-04.6.1-07 (Tampering ID-Token) | `server_metadata_url` → JWKS + Signatur-Prüfung durch authlib |
| T-04.6.1-08 (Repudiation Replay) | state/nonce session-bound, einmalig konsumiert |
| T-04.6.1-09 (Info Disclosure Secrets) | Nur `registered`/`missing` als Status geloggt, KEIN client_secret |
| T-04.6.1-10 (Open Redirect) | Nur `url_for(...)` + literal `/?modal=login`, kein `next`-Param |
| T-04.6.1-12 (Account Takeover) | Google: `email_verified` Pflicht; Microsoft: `common`-Tenant validiert serverseitig |
| T-04.6.1-13 (Session Fixation) | `session.clear()` vor `_login_user` |
| T-04.6.1-14 (Falsches Redirect-URI) | ProxyFix(x_proto=1, x_host=1) → `_external=True` baut https://app.getnerve.app/... |
