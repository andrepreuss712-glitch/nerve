## Gaps

### GAP-1: Phase-04.3-Regression entdeckt während UAT
- **Was:** `templates/base.html:75` referenzierte `g.user` ohne Guard, crashte auf Public-Pages (`/changelog/`, `/impressum`)
- **Ursache:** Wahrscheinlich Phase 04.3 Design Unification — die Verification dort hat nur Logged-In-Routes getestet, Public Pages wurden historisch nie über Flask aufgerufen (waren auf Cloudflare-Landing)
- **Fix:** `{% if g.get('user') %}` Guard um sidebar-user-block (commit 9c87fec)
- **Status:** behoben, nicht blockierend für Phase-Abschluss

### GAP-2: deploy.sh-Bug — database/ wurde nie hochgeladen
- **Was:** `deploy.sh` scp-Liste enthielt `app.py config.py requirements.txt extensions.py routes services templates static` — **aber nicht `database/`**
- **Impact:** Seit Phase 3 (Deployment) wurden Änderungen an `database/models.py` nie auf den VPS deployed. Phase 04.6.1 war die erste seit Monaten die `models.py` anfasste, daher wurde der Bug erst jetzt sichtbar
- **Fix (hot-fix):** `database` in die scp-Liste hinzugefügt (commit 91958a2)
- **Strukturelle Lücke:** Der GSD-Verifier prüft lokalen Code gegen lokalen Checkout, nicht gegen den deployten Stand. Das ist ein blinder Fleck im System
- **Empfohlene Folge-Phase:** `04.6.2-deploy-hardening`
  - `deploy.sh` auf `rsync --exclude` umstellen (ganzes Projekt minus Exclude-Liste statt explizite Datei-Enumeration)
  - Post-deploy Verifier-Check einbauen: `ssh VPS 'git log --oneline -1 /opt/nerve/app/'` vs. lokales HEAD — wenn Mismatch, fail
  - Audit: alle anderen GSD-Phasen retrospektiv prüfen ob sie Dateien in nie-hochgeladenen Ordnern berührt haben

### GAP-3: prompt=consent UX-Optimierung
- **Was:** Microsoft-Authorize-Request sendet aktuell `prompt=consent`, was jeden User bei jedem Login den Consent-Screen sehen lässt — auch wenn er schon zugestimmt hat
- **Impact:** UX-Verschlechterung nach erstem Login, kein Bug
- **Vorgehen:** `prompt=consent` nur beim allerersten Login (Register-Route) schicken, danach `prompt=select_account` oder gar nichts
- **Empfohlene Folge-Phase:** `04.6.2-oauth-polish`

### GAP-4: Landing-Seite + `getnerve.app` vs. `app.getnerve.app` Button-Navigation
- **Was:** Login-Modal wird auch auf der Cloudflare-Landing (`getnerve.app`) angezeigt und zeigt korrekt auf `https://app.getnerve.app/auth/...`. Funktioniert, aber die Architektur-Trennung Landing/App sollte dokumentiert werden.
- **Impact:** Kein Bug, nur Doku-Schuld
- **Action:** In `.planning/PROJECT.md` oder `ARCHITECTURE.md` klarstellen: `getnerve.app` = Landing auf Cloudflare, `app.getnerve.app` = Flask auf Hetzner, OAuth-Endpoints liegen nur auf `app.`

### GAP-5: `/var/www/.gunicorn` Permission-Warnung beim Start
- **Was:** `gunicorn Control server error: [Errno 13] Permission denied: '/var/www/.gunicorn'`
- **Impact:** Kosmetisch, kein Runtime-Impact
- **Fix:** `Environment=HOME=/opt/nerve` in `deploy/nerve.service` hinzufügen
- **Einordnung:** Nice-to-have für Folge-Phase
