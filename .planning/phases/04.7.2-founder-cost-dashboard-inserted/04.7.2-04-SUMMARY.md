# Phase 04.7.2 Plan 04: Admin Dashboard Shell + Übersicht + Einnahmen Summary

Blueprint `admin_dashboard_bp` unter `/admin/dashboard/`, vollständig login- und superadmin-gated, mit client-side 6-Tab-Navigation. Tabs Übersicht und Einnahmen sind voll funktional (KPI-JSON-Endpoint + Chart.js-Rendering + 4 Aggregat-Tabellen client-side geladen). Tabs Ausgaben/Kunden/EÜR/Export sind Stub-Templates bis Plans 05/06/07.

## What Was Built

### routes/admin_dashboard.py (~265 Zeilen)

| Route | Method | Auth | Returns |
|---|---|---|---|
| `/admin/dashboard/` | GET | `@login_required @superadmin_required` | HTML shell mit 6 Tabs |
| `/admin/dashboard/api/overview` | GET | gated | JSON: kpis + mrr_costs_12m + margin_12m |
| `/admin/dashboard/einnahmen` | GET | gated | JSON: summary + by_plan + by_country + transactions |

**Helpers:**
- `PERIOD_RE = re.compile(r'^\d{4}-\d{2}$')` — strikt validiert, `abort(400)` bei Format-Mismatch
- `_parse_period(s)` → `(start_date, end_date_exclusive)` mit Fallback auf aktuellen Monat
- `_mrr_from_active_orgs(db)` → Sum der `plan_preis` über alle Organisationen mit `subscription_status='active'`, Fallback auf `PLANS[org.plan]['preis']`

**Gate-Pattern:** `@login_required` nutzt Flask-session user_id → DB-Lookup → `g.user`; `@superadmin_required` prüft `g.user.is_superadmin == True`, sonst `abort(403)`. Unauthenticated → 302 Redirect zu `/login`.

### Templates (templates/admin/)

| File | Content |
|---|---|
| `dashboard.html` | base.html extension, Chart.js 4.4 CDN, 6 `<button data-tab="…">`, 6 `<section class="fcd-panel">` |
| `_tab_uebersicht.html` | 5 KPI-Karten (data-kpi="mrr/costs_mo/marge/active_users/gewinn_mo") + 2 canvas (fcd-chart-mrr-costs, fcd-chart-margin) + Breakeven-Bar |
| `_tab_einnahmen.html` | 4 Tabellen (summary, plans, country, tx) + inline `<script>` mit MutationObserver, lädt lazy wenn Tab sichtbar wird |
| `_tab_ausgaben.html` / `_kunden.html` / `_eur.html` / `_export.html` | Stub: "Wird in Plan 04.7.2-0X implementiert" |

### Static Assets

- **admin_dashboard.js**: Vanilla IIFE. Tab-switcher via `data-tab` attribute, URL-hash-sync via `history.replaceState`. `FCD.loadOverview(period)` fetch+fill KPIs, `FCD.renderMrrCosts` + `FCD.renderMarginChart` mit Chart.js. Period-picker reload on change.
- **admin_dashboard.css**: `.fcd-*` namespace, dark theme (#1C2333 card bg, #00D4AA accent, #ef4444 cost line), responsive grid (5 KPIs → 2-col, 2 charts → 1-col auf <900px).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `Organisation.plan_price_eur` existiert nicht**
- **Found during:** Task 2 Implementation
- **Issue:** Plan-Text nutzte `Organisation.plan_price_eur` für MRR-Query. Der Column existiert nicht — Organisation hat `plan_preis` (Integer EUR). Der Plan hatte diesen Hinweis bereits ("WICHTIG: prüfen").
- **Fix:** Helper `_mrr_from_active_orgs(db)` iteriert aktive Orgs, nimmt `plan_preis` und fällt auf `PLANS[org.plan]['preis']` zurück.
- **Files modified:** `routes/admin_dashboard.py`
- **Commit:** 2046e9b

**2. [Rule 1 - Bug] Off-by-one in 12-Monats-Fenster**
- **Found during:** Task 2 Implementation (Code-Review beim Schreiben)
- **Issue:** Plan-Pseudocode dekrementierte Cursor 12x und inkrementierte dann direkt in der zweiten Loop — resultierendes Fenster startete einen Monat zu früh und der Startmonat selbst war nicht enthalten.
- **Fix:** Erste Loop dekrementiert 11x, zweite Loop inkrementiert nach dem Append → Fenster [start-11M, start] inklusive.
- **Files modified:** `routes/admin_dashboard.py`
- **Commit:** 2046e9b

**3. [Rule 3 - Blocking] Fehlende `client` fixture in conftest.py**
- **Found during:** Task 1 TDD RED phase
- **Issue:** Plan erwartete `client`-Fixture in `conftest.py`, existierte nicht. Außerdem hätte `login_required` sonst seine eigene DB-Session gegen die Live-DB geöffnet → Test-User nicht auffindbar.
- **Fix:** Neue `client` fixture mit `monkeypatch.setattr(database.db, 'engine'|'SessionLocal'|'db_session', ...)` auf in-memory SQLite. `app` Import **nach** dem Patching, Routes nutzen `get_session()` → `SessionLocal()` lazily, bekommen automatisch die gepatchte Session.
- **Files modified:** `tests/conftest.py`
- **Commit:** 2046e9b

**4. [Rule 1 - Bug] Plan-Test nutzte nicht-existierende Felder**
- **Found during:** Task 1 TDD RED phase
- **Issue:** Plan-Test-Code nutzte `Organisation(..., plan_key='solo', ...)`. `plan_key` existiert aber nur auf `RevenueLog`, nicht auf `Organisation` (die hat `plan`).
- **Fix:** Test-Helper `_make_org_and_user()` nutzt `plan='starter'`. `onboarding_done=True` auf User (nicht Org) gesetzt, damit `login_required` nicht zu `/onboarding` redirected.
- **Files modified:** `tests/test_admin_dashboard_auth.py`
- **Commit:** 2046e9b
