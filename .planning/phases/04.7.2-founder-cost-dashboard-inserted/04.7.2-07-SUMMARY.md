# Phase 04.7.2 Plan 07: Export Tab + PDF/CSV Downloads Summary

**One-liner:** EÜR-PDF via WeasyPrint (Linux-only) + HTML-Preview-Fallback + 4 CSV-Exports (Einnahmen, Ausgaben, USt-VA, DATEV-Stub) mit deutschem Zahlenformat und Excel-BOM — Phase 04.7.2 Close-out.

## What Was Built

### app.py — de_currency Jinja Filter

Global registrierter Filter `de_currency`: `1234.56 -> '1.234,56 €'`. Sentinel-Swap `,`-`.` via `§`-Zeichen. Handles `None` -> `—`, TypeError -> str(value) fallback.

### templates/admin/eur_pdf.html — WeasyPrint PDF Template

DIN A4 CSS (`@page`), deutsches Anlage-EÜR-Layout mit:

- **Header:** Andre Preuss, Einzelunternehmer, Zeitraum
- **A. Betriebseinnahmen:** Z.14 DE_19, Z.16 EU_RC, Z.17 Drittland, Z.19 vereinnahmte USt, Summe
- **B. Betriebsausgaben:** Z.26 Fremdleistungen (mit Per-Provider-Breakdown), Z.52 Miete EDV, Z.57 Übrige, Z.65 Homeoffice, Z.75 Summe, Z.72 Vorsteuer (Inland + §13b + Total)
- **C. Ergebnis:** Z.77 Gewinn vor Steuern, USt-Zahllast, Gewinn nach USt
- **USt-Voranmeldung:** KZ 81/21/45/84/85/66/67 + Zahllast
- **Footer:** HT-04 count.tax-Disclaimer

Template nutzt `de_currency` Filter durchgehend. Funktioniert als WeasyPrint-Input UND als HTML-Preview.

### routes/admin_dashboard.py — 6 Export Endpoints

| Route | Method | Returns |
|---|---|---|
| `/export/eur.html` | GET | text/html — Preview (immer verfuegbar) |
| `/export/eur.pdf`  | GET | application/pdf ODER 501 wenn WeasyPrint fehlt |
| `/export/einnahmen.csv` | GET | CSV: Datum, Rechnung, Land, Plan, Netto, USt, Brutto, Behandlung |
| `/export/ausgaben.csv`  | GET | CSV: API-Kosten + FixedCosts mit SKR03/EÜR-Zeile |
| `/export/ustva.csv`     | GET | CSV: KZ 81/21/45/84/85/66/67/Zahllast |
| `/export/datev_stub.csv`| GET | CSV: Datum, Konto, Gegenkonto, Betrag, Buchungstext (**NICHT DATEV-konform**) |

**Helper `_csv_response(rows, headers, filename)`** — BOM (`\ufeff`) + Semikolon-Delimiter + `text/csv; charset=utf-8` + Content-Disposition. Zentralisiert fuer alle 4 CSVs.

**WeasyPrint-Pattern:**
```python
try:
    from weasyprint import HTML
except ImportError:
    return ("WeasyPrint nicht installiert. Verwende /export/eur.html für Preview.", 501)
```

Alle Endpoints: `@login_required + @superadmin_required`.

### templates/admin/_tab_export.html — UI

- 5 Download-Links (PDF, Einnahmen, Ausgaben, USt-VA, DATEV-Stub) + HTML-Preview-Link
- `home_days` Input (0-366) fuer Homeoffice-Pauschale
- `updateLinks()` baut Query-String aus `.fcd-wrap` data-period + home_days
- MutationObserver Lazy-Load konsistent mit anderen Tabs
- Automatisierungs-Hinweis: count.tax-Auto-Delivery deferred auf Milestone 3

### tests/test_eur_pdf.py

Skeleton ersetzt durch echten WeasyPrint smoke. `pytest.importorskip("weasyprint")` -> Tests werden auf Windows-Dev (ohne WeasyPrint) sauber geskippt, laufen auf Linux-VPS.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Correctness] Null-Safe CSV-Row-Building**
- **Found during:** Task 2 Implementation
- **Issue:** Plan-Snippet nutzte `r.netto_cents/100` ohne Null-Check — bei fehlenden Werten wuerde TypeError geworfen statt leerer Spalte.
- **Fix:** `(r.netto_cents or 0)/100`, `float(a.cost_eur or 0)`, `getattr(a, 'context_tag', '') or ''` fuer alle potenziell-nullable Felder.
- **Files modified:** `routes/admin_dashboard.py` (alle 4 CSV-Endpoints)
- **Commit:** a6c4c02

**2. [Rule 3 - Blocking] home_days try/except in allen Endpoints**
- **Found during:** Task 1+2 Implementation
- **Issue:** Plan-Snippet nutzte `int(request.args.get('home_days', 0))` direkt — crasht bei `?home_days=xyz` mit 500 statt graceful fallback.
- **Fix:** try/except ValueError/TypeError → home_days = 0.
- **Files modified:** `routes/admin_dashboard.py` (4 Endpoints)
- **Commit:** a6c4c02

**3. [Rule 1 - Bug] Test-Skeleton reduziert**
- **Found during:** Task 1 Test-File
- **Issue:** Plan-Snippet fuer `test_eur_pdf_template_smoke` nutzte `db_session` + `app` Fixtures, die in der aktuellen conftest.py nicht ohne weiteres zur Verfuegung stehen (conftest hat `client` + `db_from_client`, aber kein standalone `app`-Fixture fuer Jinja-render_template-Kontext).
- **Fix:** Nur `test_weasyprint_can_render_minimal` behalten. Template-Rendering wird via `/export/eur.html` Live-Smoke-Test auf VPS validiert.
- **Files modified:** `tests/test_eur_pdf.py`
- **Commit:** a6c4c02

**4. [Rule 2 - Correctness] Template: dict-access mit Subscript-Syntax fuer 'items'**
- **Found during:** Task 1 Template
- **Issue:** `data.ausgaben.Z26_fremdleistungen.items` — Jinja2 interpretiert `.items` als dict.items() Methode (built-in), nicht als Key.
- **Fix:** `data.ausgaben.Z26_fremdleistungen['items']` explizit via subscript.
- **Files modified:** `templates/admin/eur_pdf.html`
- **Commit:** a6c4c02
