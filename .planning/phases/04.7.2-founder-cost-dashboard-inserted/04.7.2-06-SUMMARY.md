# Phase 04.7.2 Plan 06: EÜR Calculator §13b Summary

**One-liner:** TDD-implementierter EÜR-Calculator mit Anlage-EÜR-Zeilen-Mapping und §13b Reverse-Charge für US-Drittland-API-Provider, inkl. Tab EÜR Rendering.

## Was wurde gebaut

Der steuerkritische Kern der Phase: `services/eur_calculator.compute_eur(start, end, db, home_days=0)` liefert ein strukturiertes Dict mit allen relevanten Anlage-EÜR-Zeilen (Z.14, Z.16, Z.17, Z.19, Z.26, Z.52, Z.57, Z.65, Z.72, Z.75, Z.77) sowie einem kompletten USt-Voranmeldung-Block (KZ 81, 21, 45, 84, 85, 66, 67, zahllast).

### §13b Reverse-Charge Formel

Für US-Drittland-API-Provider (anthropic, deepgram, elevenlabs):

```
KZ84 (Bemessungsgrundlage) = Σ netto Kosten anthropic + deepgram + elevenlabs
KZ85 (USt §13b)            = KZ84 × 0.19
KZ67 (Vorsteuer §13b)      = KZ85          # harter Invariant — identisch
Netto-Effekt §13b auf Zahllast = KZ85 − KZ67 = 0
```

### USt-Zahllast Formel

```
ust_zahllast = (KZ81.ust + KZ85) − (KZ66 + KZ67)
             = (vereinnahmte USt + RC-USt) − (Inlands-VSt + RC-VSt)
```

### Zeilen-Mapping

| Zeile | Quelle | Inhalt |
|-------|--------|--------|
| Z.14  | RevenueLog tax_treatment='DE_19' | Umsatzerlöse 19% USt DE |
| Z.16  | RevenueLog tax_treatment='EU_RC' | EU B2B Reverse Charge (Ausgangsleistung) |
| Z.17  | RevenueLog tax_treatment='DRITTLAND' | Nicht steuerbare Drittland-Leistungen |
| Z.19  | Σ Z14.ust | Vereinnahmte Umsatzsteuer |
| Z.26  | ApiCostLog provider ∈ RC_13B_PROVIDERS | Bezogene Fremdleistungen (US-API-Provider) |
| Z.52  | FixedCost eur_line=52 | Miete/Leasing EDV (Hetzner) |
| Z.57  | FixedCost eur_line=57 + Stripe-Gebühren | Übrige Betriebsausgaben |
| Z.65  | FixedCost cycle='per_day' × home_days | Häusliches Arbeitszimmer |
| Z.72  | Inland-VSt + KZ67 | Vorsteuer gesamt |
| Z.75  | Σ Z26+Z52+Z57+Z65 | Summe Betriebsausgaben (netto) |
| Z.77  | Z19-Einnahmen − Z75 | Gewinn vor Steuern |
