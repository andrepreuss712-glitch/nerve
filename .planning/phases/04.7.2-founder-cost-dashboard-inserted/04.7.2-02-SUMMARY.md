# Phase 04.7.2 Plan 02: API Cost Tracking Hooks Summary

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] `SessionLocal` module-level import breaks monkeypatching**
- **Found during:** Task 1 test development
- **Issue:** Plan spec imported `from database.db import SessionLocal` inside `log_api_cost`. `monkeypatch.setattr('database.db.SessionLocal', ...)` would then not affect an already-bound local name inside the try block.
- **Fix:** Changed to `import database.db as _db_mod` then `_db_mod.SessionLocal()`. Tests use `monkeypatch.setattr(_db_mod, 'SessionLocal', ...)`.
- **Files modified:** `services/cost_tracker.py`, `tests/test_cost_tracker.py`
- **Commit:** 6e6ef56

**2. [Rule 3 - Blocking] `db_session` fixture uses in-memory engine not shared with `SessionLocal`**
- **Found during:** Task 1 test development
- **Issue:** `tests/conftest.py` creates a fresh in-memory engine for `db_session`, while `SessionLocal` in `database.db` points at the real SQLite file. Tests would write to different DBs.
- **Fix:** Added `patched_sessionlocal` fixture that rebinds `SessionLocal` onto `db_session.get_bind()` via `sessionmaker`.
- **Files modified:** `tests/test_cost_tracker.py`
- **Commit:** 6e6ef56

**3. [Rule 2 - Correctness] Deepgram `_cost_opened_at` stored outside `_deepgram_sessions`**
- **Issue:** Plan suggested modifying `session_dict['_cost_opened_at']`, but `_deepgram_sessions[sid] = connection` stores the raw SDK connection object, not a dict.
- **Fix:** Added parallel `_cost_opened_at = {}` module dict guarded by the existing `_sessions_lock`. Set in `_open_deepgram_connection`, popped in `_close_deepgram_connection`.
- **Files modified:** `services/deepgram_service.py`
- **Commit:** b46434f

**4. [Rule 2 - Correctness] TTS hook moved inside status_code==200 branch**
- **Issue:** Plan showed the hook after the `try:` block, which would fire even on HTTP errors where `response` might not be defined or `response.status_code != 200`.
- **Fix:** Moved hook inside the `if response.status_code == 200:` branch before `return response.content`. This also means failed TTS calls don't generate phantom cost rows.
- **Files modified:** `services/training_service.py`
- **Commit:** b46434f
