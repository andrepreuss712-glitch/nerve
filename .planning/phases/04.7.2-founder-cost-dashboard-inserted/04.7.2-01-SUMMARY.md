## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `__table_args__` assignment after class definition does not register UniqueConstraint**
- **Found during:** Task 3 (test_exchange_rate_uniqueness_constraint failed)
- **Issue:** Plan listed `ExchangeRate.__table_args__ = (...)` and `ApiRate.__table_args__ = (...)` as post-class assignments. SQLAlchemy reads `__table_args__` only during class body declarative processing, so the assignment did not actually register the UniqueConstraint with the mapper.
- **Fix:** Moved `__table_args__ = (UniqueConstraint(...),)` inside the class body of `ApiRate` and `ExchangeRate`. Removed the post-class assignment block.
- **Files modified:** `database/models.py`
- **Commit:** 54ce7ee

### Other Notes
- `Numeric` was missing from the existing `from sqlalchemy import …` line in `database/models.py` and was added (planned). `Date`, `Text`, `UniqueConstraint` were already present.
- `init_db(engine)` is already called at app startup before `_migrate()`, so no changes were needed to `_migrate()` itself — the 6 new tables get auto-created via existing `Base.metadata.create_all`.
