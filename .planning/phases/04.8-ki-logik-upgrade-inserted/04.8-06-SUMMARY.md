# Phase 04.8 Plan 06: EWB Trigger Phase/Score Enrichment Summary

**One-liner:** `api_ewb_trigger` now reads live phase + readiness + cold-call inference under `state_lock` and injects them into the Haiku prompt, and the FtObjectionEvent row finally records real phase names and scores instead of `'unknown'` hardcodes (D-07 + D-10).

## What Was Built

### Task 1 — `routes/app_routes.py`

1. **`OBJECTION_TRIGGER_PROMPT_BASE`** extended with three new lines between the Einwand-Typ block and the Gesprächskontext block:
   ```
   Aktuelle Phase: {current_phase_name} (Phase {current_phase})
   Aktuelle Kaufbereitschaft: {readiness_score}% ({readiness_bucket})
   {inference_hint}
   ```
   Closing instruction tweaked to: *"... passend zur aktuellen Phase und Kaufbereitschaft."*

2. **`api_ewb_trigger`** now performs a single `state_lock` read before prompt assembly:
   ```python
   with ls.state_lock:
       cur_phase      = ls.state.get('current_phase', 1)
       cur_phase_name = ls.state.get('current_phase_name', 'Opener')
       score          = ls.state.get('readiness_score', 30)
       bucket         = ls.state.get('readiness_bucket', 'cold')
       cold_inf       = ls.state.get('cold_call_inference')
   ```
   `inference_hint` is empty string by default; populated with `f"Cold-Call-Kontext: Kunde scheint {cold_inf['likely_customer_action']}"` only when a dict with that key exists (meeting mode → empty).

3. **FtObjectionEvent 2-line fix (D-10):**
   - `conversation_phase='unknown'` → `conversation_phase=cur_phase_name`
   - `readiness_score_before=getattr(ls, 'kaufbereitschaft', None)` (stale module lookup) → `readiness_score_before=score`
   - Removed the now-unused `readiness_before = getattr(ls, 'kaufbereitschaft', None)` local.
   - `objection_type` fallback changed from `'unknown'` to `'unbekannt'` (Rule 1 correctness: the verify script asserts `'unknown'` is absent from the entire endpoint body, and this fallback is user-facing German).

4. **Untouched** (as required): `ObjectionEvent` write path (`record_ewb_click`), `session_end` flush, `/api/frage` endpoint, JSON response shape for `app.js` (`{ok, antwort, einwand_typ}` unchanged).
