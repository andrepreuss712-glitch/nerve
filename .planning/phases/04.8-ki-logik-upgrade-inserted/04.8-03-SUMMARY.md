# Phase 04.8 Plan 03: Cold-Call Inference Haiku Helper Summary

**One-liner:** Haiku-backed cold-call customer-state inference (D-05) with strict mode guard — meeting mode never triggers a Haiku call, cold_call mode populates `state['cold_call_inference']` on the same every-5th-cycle cadence as the phase classifier.

## What Was Built

### Task 1 — Haiku helper + wrapper + analyse_loop integration

**`services/claude_service.py`:**
- **`COLDCALL_INFER_PROMPT`** — German few-shot prompt from RESEARCH §3.2 with 5 inference rules (seller repeats → customer zögert; seller sagt "verstehe" → Einwand; etc.) and strict JSON-only output format.
- **`infer_customer_state(seller_transcript, phase) -> dict | None`** — Haiku call (model `claude-haiku-4-5-20251001`, `max_tokens=120`). Formats last 6 seller lines, parses JSON with markdown-fence tolerance (matching `classify_phase` pattern), validates `likely_customer_action` is a string and `confidence ∈ [0,1]`, caps strings at 200 chars, stamps ISO UTC `ts`. Wired to cost tracker with `context_tag='coldcall_infer'`. Returns `None` on empty input or any parse/API failure — never raises.
- **`analyse_loop` integration** — new block nested INSIDE the existing `if _phase_cycle_counter % 5 == 0:` branch from P02, AFTER the phase-classifier state writes. Reads `mode` + `current_phase` under `state_lock`; when `mode == 'cold_call'`, pulls last 6 lines from `ls.analysiert_bisher` under `buffer_lock`, calls `infer_cold_call_context(..., haiku_caller=infer_customer_state)`, writes result (or `None`) to `ls.state['cold_call_inference']` under `state_lock`. Wrapped in try/except — errors never crash analyse_loop or the phase-classifier block.

**`services/ki_logik.py`:**
- **`infer_cold_call_context(seller_transcript, current_phase, mode, haiku_caller=None)`** — replaces the `NotImplementedError` stub. Thin mode-guard wrapper:
  - `mode != 'cold_call'` → return `None` immediately (zero cost in meeting mode)
  - `haiku_caller is None` → return `None` (defensive; tests exercise this path)
  - otherwise delegate to `haiku_caller(seller_transcript, current_phase)` and return its result verbatim
- Injectable `haiku_caller` avoids the claude_service ↔ ki_logik import cycle and makes the wrapper trivially unit-testable without mocking the Anthropic SDK.

### Task 2 — Tests (`tests/services/test_ki_logik.py`)

TDD flow — 3 new tests appended after the existing dynamic_ewb_buttons block:

- **`test_infer_cold_call_meeting_mode_skips`** — `mode='meeting'` returns `None` even when a working `haiku_caller` is passed (proves the guard fires BEFORE the caller is invoked).
- **`test_infer_cold_call_without_haiku_caller`** — `mode='cold_call'` with no caller returns `None` (defensive path).
- **`test_infer_cold_call_invokes_haiku_caller`** — `mode='cold_call'` with a fake caller: asserts the returned dict is passed through AND the caller received `(seller_transcript, current_phase)` as positional args in the correct order.

## STRIDE Mitigations Implemented

- **T-04.8.3-01 (Info Disclosure — coldcall inference prompt):** Only seller transcript is ever sent to Haiku (already legal basis Art 6 f for cold calls). Meeting mode skips the Haiku call entirely — zero data leaves the server in meeting mode for this feature. Inference results held in-memory only; NOT routed through `_write_ft_assistant_event` so the existing cold_call DSGVO cutoff in that function is not bypassed.
- **T-04.8.3-02 (Tampering — Haiku JSON → state):** Strict parse (`json.loads`) inside try/except, type-check `likely_customer_action` is `str`, range-check `confidence ∈ [0,1]`, length caps of 200 chars on both string fields before any state write. Invalid/malformed responses return `None`.
- **T-04.8.3-03 (DoS — inference cost):** Nested inside the existing 5-cycle throttle from P02 → ~6 Haiku inference calls per minute in cold_call mode, 0 in meeting mode. Seller window capped at 6 lines → bounded input tokens. `max_tokens=120` → bounded output cost.
- **T-04.8.3-04 (Repudiation):** Accepted per plan — pre-call feature, button-click FT logging handled by P06.
