# Phase 04.8 Plan 01: KI-Logik Foundation Summary

**One-liner:** Pure-Python primitives for deterministic readiness scoring, hint prioritization, and phase-aware EWB buttons, plus pytest scaffolding and the Phase 04.8 state schema — foundation for all subsequent 04.8 plans, zero Haiku calls, zero loop integration.

## What Was Built

- **`services/ki_logik.py`** (~135 lines): Three pure functions — `compute_readiness_score(state, transcript_window) -> (int, bucket)`, `select_active_hint(candidates) -> dict|None`, `dynamic_ewb_buttons(phase, base_buttons) -> list`. Module-level constants `SCORE_FACTORS`, `SCORE_BASE`, `BUCKETS`, `PHASE_BUTTONS`, `HINT_PRIORITY`. Stubs for `detect_phase` and `infer_cold_call_context` raising `NotImplementedError` (P02/P03 implements).
- **User override documented in code:** `SCORE_FACTORS['einwand_geloest'] = 20` with inline comment explaining divergence from briefing's +15.
- **Test scaffolding:** `tests/services/__init__.py`, `tests/services/test_ki_logik.py` (22 tests, all green in 0.04s). Extended `tests/conftest.py` with `sample_state` fixture (factory pattern, allows per-test overrides) and repo-root sys.path insertion so `from services.ki_logik import …` works regardless of pytest cwd.
- **State schema extended:** `services/live_session.py` `state` dict now includes 11 new Phase 04.8 keys (`current_phase`, `current_phase_name`, `phase_confidence`, `phase_changed_at`, `phase_change_count`, `readiness_score`, `readiness_bucket`, `score_factors_seen`, `active_hint`, `ewb_buttons`, `cold_call_inference`). `reset_session()` clears all 11 under `state_lock`.

## Test Coverage (22/22 pass)

- `compute_readiness_score`: base case, accumulation to closing, einwand_geloest=+20 explicit + applied, negative factors, clamp to 0, clamp to 100, bucket boundaries (30→cold, 35→warm, 60→warm, 75→hot, 85→closing).
- `select_active_hint`: empty list, all None, priority-1 wins, tie → first wins, ignores falsy/missing-priority entries.
- `dynamic_ewb_buttons`: phase 1 returns 6 buttons, all phases 1-6 have ≥5 buttons, unknown phase falls back to base, unknown phase with no base → `[]`, returns fresh list (mutation isolation).
