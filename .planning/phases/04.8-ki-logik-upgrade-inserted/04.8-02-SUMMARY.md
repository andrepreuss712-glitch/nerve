# Phase 04.8 Plan 02: Phase Classifier + Hysteresis Summary

**One-liner:** Haiku-backed phase classifier (every 5th cycle) feeding deterministic hysteresis rule in `ki_logik.detect_phase` — 6-phase live signal with flicker suppression, Phase-5 regression allowed only with strong evidence + 3-cycle debounce; rank_ewb Haiku call deleted (D-08 cost save).

## What Was Built

### Task 1 — classify_phase + rank_ewb deletion (`services/claude_service.py`)

- **`PHASE_CLASSIFIER_PROMPT`** constant (German, 6 phases + current context + transcript window).
- **`classify_phase(transcript_window, current_phase, elapsed_s, mode)`** — Haiku call (model `claude-haiku-4-5-20251001`, `max_tokens=60`). Validates `1<=phase<=6`, `0<=conf<=1`, strips markdown fences, returns `{'phase','confidence','grund'}` or `None`. Wired to cost tracker with `context_tag='phase_classify'`.
- **`_PHASE_NAMES`** module dict (1=Opener ... 6=Abschluss).
- **`analyse_loop` integration block** after the main Einwand-Analyse Haiku call:
  - `_phase_cycle_counter` stored as attribute on the `analyse_loop` function object (avoids new module global).
  - Every 5th cycle, pulls last 10 lines from `ls.analysiert_bisher` (under `buffer_lock`), reads `cur_phase / phase_change_count / last_change_cycle / mode` (under `state_lock`), calls `classify_phase`, then `ki_logik.detect_phase` for hysteresis.
  - On accepted change: writes `current_phase`, `current_phase_name`, `phase_changed_at` (ISO UTC), `phase_change_count`, `_phase_cycle_at_last_change`, plus `phase_confidence` under `state_lock`. Log line shows transition + reason.
  - Wrapped in try/except — loop errors never crash analyse_loop.
- **DELETED** per D-08 + user override #2:
  - `rank_ewb()` function
  - `EWB_RANKING_PROMPT_BASE`
  - `_ewb_rank_counter` module global
  - The `if _ewb_rank_counter % 3 == 0` ranking block in `analyse_loop`

### Task 2 — detect_phase hysteresis + tests (`services/ki_logik.py`, `tests/services/test_ki_logik.py`)

- **Replaced** the `NotImplementedError` stub with the pure-Python hysteresis rule from RESEARCH §Q4. Signature changed to `detect_phase(raw_phase, raw_confidence, current_phase, phase_change_count=0, cycles_since_change=0) -> (int, float)`.
- Rules implemented:
  - `raw == current` → pass-through
  - `raw > current` → accept only if `confidence >= 0.7`
  - `raw < current` → accept only if `raw == 5 AND confidence >= 0.8 AND cycles_since_change >= 3 AND current_phase >= 2`
  - all other regressions → blocked
- **10 new tests** in `test_ki_logik.py::test_detect_phase_*`:
  - same-phase pass-through
  - forward low-conf blocked / high-conf advances / multi-skip allowed
  - regress-to-5 allowed with debounce (from 4, from 6)
  - regress-to-5 debounce blocks (cycles<3)
  - regress-to-5 low-conf blocks
  - regress-to-non-5 always blocked
  - regress-from-phase-6-to-3 blocked

- **Issue:** 3 tests used `current_phase=4` for "regression to 5" cases, but 5>4 is a forward advance, not regression. The hysteresis rule for regression only triggers when `raw_phase < current_phase`.
- **Fix:** Rewrote those tests to use `current_phase=6` (6→5 is a true regression). Also replaced the "regress from phase 1" test with "regress from phase 6 to 3 blocked" — more semantically meaningful because phase-1 can only go forward, so a regression test from phase 1 is degenerate.
- **Files modified:** tests/services/test_ki_logik.py
- **Commit:** 2a37526

**3. [Rule 1 - Style] _phase_cycle_counter on function object instead of module global**
- **Found during:** Task 1 integration block.
- **Issue:** Plan example uses `getattr(analyse_loop, '_phase_cycle_counter', 0) + 1`. That's already the function-attribute pattern (not a new module global), which is cleaner than adding another `global` statement. Kept plan's approach.
