**Current state per minute (continuous speech, 30 cycles):**

| Call | Freq/min | Model | In tok | Out tok | In $/call | Out $/call | $/min |
|------|---------|-------|--------|---------|-----------|------------|-------|
| Einwand-Analyse | ~30 | Haiku 4.5 | ~500 | ~200 | — | — | — |
| EWB-Ranking | ~10 | Haiku 4.5 | ~200 | ~30 | — | — | — |

**Haiku 4.5 pricing (as of late 2025)** `[ASSUMED — must be verified before planner locks]`:
- Input: ~$1 / 1M tokens
- Output: ~$5 / 1M tokens

Back-of-envelope with the above token estimates:
- Einwand-Analyse: (30 × 500 × $1 + 30 × 200 × $5) / 1M = $0.015 + $0.030 = **$0.045/min**
- EWB-Ranking: (10 × 200 × $1 + 10 × 30 × $5) / 1M = $0.002 + $0.0015 = **$0.0035/min**
- **Total today: ~$0.049/min ≈ $0.73 / 15-min call**

**New calls proposed for 04.8:**

| Call | Proposed freq | In tok | Out tok | $/min (est) |
|------|--------------|--------|---------|-------------|
| Phase-Classifier (D-01) | every 5th cycle (~6/min) | ~600 | ~40 | $0.0048 |
| Cold-Call-Inferenz (D-05) | every 5th cycle, coldcall only (~6/min) | ~400 | ~80 | $0.0048 |

**Projected total (both modes):** ~$0.054/min. Cold-call worst case: ~$0.059/min → **~$0.88 / 15-min call.**

**Verdict:** Budget is comfortable. Recommendation:
- **Phase classifier: every 5 cycles (~10s)** as default. Bursts to every 2 cycles if phase_confidence < 0.6 for 2 consecutive classifications. Debounce phase changes via hysteresis (see Q4).
- **Cold-call inference: every 5 cycles**, synced with phase classifier so they hit on the same cycle (reduces scheduling jitter).
- Both calls should be **skipped when `buffer_lock` transcript buffer is empty** (no new speech = no new info).

### Q4 — Phasen-Hysterese

**Problem:** Phase 5 (Einwandbehandlung) can legitimately recur after Phase 4 (Pitch). Pure linear progression breaks this. But also: jitter (classifier oscillating between Phase 3 and Phase 4) must be suppressed.

**Proposed rule set in `detect_phase`:**

```
IF new_phase == current_phase:
    return (current_phase, confidence)  # no change

IF new_phase > current_phase:
    # Forward progression — require confidence ≥ 0.7
    IF confidence < 0.7: return (current_phase, confidence)
    return (new_phase, confidence)

IF new_phase < current_phase:
    # Regression — only allowed if target is Phase 5 (Einwandbehandlung)
    # AND confidence ≥ 0.8 AND at least 3 cycles since last phase change
    IF new_phase == 5 AND confidence ≥ 0.8 AND cycles_since_change ≥ 3:
        return (5, confidence)  # legitimate objection handling
    return (current_phase, confidence)  # suppress regression
```

Phase 5 → Phase 4 (or 6) is allowed as forward movement. Phase 5 can re-enter from any phase ≥ 2. Phase 1 → Phase 1 cannot regress (call start is monotonic).

`phase_changed_at` updated only on actual change. `cycles_since_change` derived from `phase_changed_at` vs current cycle counter.

function renderActiveHint(hint) {
  const slots = [
    document.getElementById('active-hint-slot'),
    document.getElementById('kp-active-hint'),
    document.getElementById('pip-active-hint'),
  ].filter(Boolean);
  if (!hint) {
    slots.forEach(s => { s.style.display='none'; s.innerHTML=''; });
    return;
  }
  const html = `<div class="hint-${hint.type}">
    <span class="hint-label">${hint.color === 'red' ? '⚠' : hint.color === 'gold' ? '★' : '●'}</span>
    <span class="hint-text">${escHtml(hint.text)}</span>
  </div>`;
  slots.forEach(s => { s.innerHTML=html; s.style.display='flex'; });
}

function renderPhaseBadge(phase, name, confidence) {
  const el = document.getElementById('phaseBadge');
  if (!el) return;
  el.textContent = `Phase ${phase} · ${name}`;
  el.classList.toggle('low-conf', confidence < 0.6);
}

function renderReadinessBadge(score, bucket) {
  const el = document.getElementById('readinessBadge');
  if (!el) return;
  el.textContent = `${score}%`;
  el.className = `readiness-badge bucket-${bucket}`;
}
