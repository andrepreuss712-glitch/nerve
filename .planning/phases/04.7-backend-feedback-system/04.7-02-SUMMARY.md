# Phase 04.7 Plan 02: Audit-Log Summary

**One-liner:** Unveraenderliches DSGVO-Audit-Log mit SQLite RAISE(ABORT)-Trigger als Defense-in-Depth, `log_action()` Helper und Wire-up in Password-Login, OAuth-Login, Logout, Session-Ende und Profil-Update.

## What Was Built

### Task 1 ‚Äî AuditLog Modell + Immutable-Trigger + log_action Helper (commit 75ed819)

- `database/models.py`: `class AuditLog(Base)` mit Feldern `user_id`, `org_id`, `action`, `target_type`, `target_id`, `details` (JSON-Text), `ip_address`, `user_agent`, `created_at`
- `services/audit.py` (neu): `log_action()` Helper ‚Äî schreibt Eintrag, faengt alle Exceptions ab damit Audit den Request nie killt
- `app.py`: SQLite Trigger `audit_log_no_update` + `audit_log_no_delete` via `RAISE(ABORT, 'audit_log is immutable')` ‚Äî idempotent via `CREATE TRIGGER IF NOT EXISTS`, nach `_migrate()` installiert
- `app.py`: `Bootstrap4Theme()` statt `template_mode='bootstrap4'` ‚Äî flask-admin 2.x API-Fix (Rule 1)

### Task 2 ‚Äî Wire-up in Auth + Session + Profil Routes (commit 65bdec9)

- `routes/auth.py`: `log_action(..., 'login', details={'method':'password'})` in `_do_login` nach `_login_user`; `log_action(..., 'logout')` in `logout()` vor `session.clear()` (liest `user_id` aus Flask-session)
- `routes/oauth.py`: `log_action(..., 'login', details={'method': provider})` in `_oauth_login_or_create` fuer beide Pfade (bestehender User + Neuanlage)
- `routes/app_routes.py`: `session_start` + `session_end` in `api_beenden` nach `db_conv.commit()` ‚Äî nur Aggregate (`mode`, `dauer_sekunden`, `einwaende_total`, `einwaende_ok`), kein Transkript (DSGVO D-07)
- `routes/profiles.py`: `log_action(..., 'profile_update', details={'name': p.name})` in `bearbeiten` nach `db.commit()`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Flask-Admin 2.x API: `template_mode` -> `Bootstrap4Theme()`**
- **Found during:** Task 1 Smoke Test
- **Issue:** `flask-admin==2.0.2` hat `template_mode` Parameter nicht mehr in `Admin.__init__()` ‚Äî wirft `TypeError: Admin.__init__() got an unexpected keyword argument 'template_mode'`
- **Fix:** `from flask_admin.theme import Bootstrap4Theme` importiert, `theme=Bootstrap4Theme()` statt `template_mode='bootstrap4'`
- **Files modified:** `app.py`
- **Commit:** 75ed819

**2. [Rule 3 - Blocking] flask-admin nicht installiert**
- **Found during:** Task 1 Smoke Test
- **Issue:** `ModuleNotFoundError: No module named 'flask_admin'` ‚Äî war in `requirements.txt` gepinnt aber nicht in venv installiert
- **Fix:** `pip install flask-admin==2.0.2` (zog auch `wtforms==3.2.1` mit)
- **Commit:** kein separater Commit (Infra-Fix, kein Code-Aenderung)

**3. [Architectural Adaptation] session_start Placement**
- **Issue:** Plan fordert `log_action` in `api_start_session` ‚Äî kein solcher HTTP-Endpoint existiert. Session-Start laeuft via Socket.IO `start_live_session` Event in `deepgram_service.py` (kein `g.user`/`g.org` verfuegbar dort).
- **Fix:** `session_start` + `session_end` beide in `api_beenden` nach `ConversationLog.commit()` geschrieben. `session_start` hat `details={'mode': session_mode}`, `session_end` hat Aggregate. Beide haben `conv.id` als `target_id`. Kein Informationsverlust ‚Äî alle Felder korrekt.
- **Files modified:** `routes/app_routes.py`
