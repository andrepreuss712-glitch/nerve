# Phase 04.7 Plan 04: Feedback-System (User-Seite) Summary

**One-liner:** User-seitiges Feedback-System mit grueenem Sidebar-Button, Modal (Typ + Text + Screenshot, 5 MB Cap, MIME-Whitelist), Quick-Rating-Endpoint und separater `feedback` DB-Tabelle unabhaengig von `FeedbackEvent`.

## What Was Built

### Task 1 — Feedback Modell + Service + Route + MAX_CONTENT_LENGTH (commit 643fd9d)

- `database/models.py`: `class Feedback(Base)` mit `typ`, `text`, `screenshot_path`, `context_url`, `status`, `kategorie`, `rating`, `notification_sent`, `created_at`, `updated_at` — komplett getrennt von `FeedbackEvent`
- `services/feedback_service.py` (neu): `save_screenshot()` mit MIME + Extension-Whitelist, UUID-Dateiname, `_ensure_dir()` fuer `/opt/nerve/uploads/feedback/`; `create_feedback()` fuer DB-Insert
- `routes/feedback.py` (neu): `POST /api/feedback` (Multipart, mit optionalem Screenshot) + `POST /api/feedback/quick` (JSON, Rating 1-5) — beide `@auth_required` via `g.user` Check
- `app.py`: `MAX_CONTENT_LENGTH = 5 * 1024 * 1024`, `feedback_bp` vor `auth_bp` registriert, DB-Migration fuer `feedback`-Tabellen-Spalten (idempotent)

### Task 2 — Sidebar-Button + Modal + JS Upload-Logik (commit 18a0fa8)

- `static/feedback.css` (neu): `.nerve-fb-btn` gruener Pulsing-Glow-Button, `.nerve-fb-modal` Fixed-Overlay, `.nerve-fb-card` Dark-Card, Formular-Styles konsistent mit NERVE Design-System
- `static/feedback.js` (neu): IIFE-Modul — Modal oeffnen/schliessen, `FormData`-Submit an `/api/feedback` via `fetch`, 3s Toast-Bestaetigung, Klick-auf-Backdrop schliesst Modal
- `templates/base.html`: Sidebar-Button (`{% if g.user %}`) nach dem flex-Spacer vor dem User-Dropdown; Modal + `feedback.css` + `feedback.js` direkt vor `</body>` (nur wenn `g.user`)
