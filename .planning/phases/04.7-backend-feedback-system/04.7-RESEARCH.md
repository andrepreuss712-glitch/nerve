### Pattern 1: `@superadmin_required` Decorator

```python
# services/auth_decorators.py (oder in bestehender auth.py anfügen)
from functools import wraps
from flask import g, abort

def superadmin_required(f):
    @wraps(f)
    def wrapper(*args, **kwargs):
        if not getattr(g, 'user', None) or not g.user.is_superadmin:
            abort(403)
        return f(*args, **kwargs)
    return wrapper
```

**ENV-Seed im bestehenden `_login_user`:**
```python
# routes/auth.py im _login_user nach erfolgreichem Login
import os
superadmin_email = os.environ.get('SUPERADMIN_EMAIL', '').strip().lower()
if superadmin_email and user.email.lower() == superadmin_email and not user.is_superadmin:
    user.is_superadmin = True
    db.commit()
```
Idempotent: einmal True, bleibt True.

### Pattern 2: Immutable Audit-Log

**Empfehlung: Defense-in-Depth — App-Layer Enforcement + SQLite-Trigger.**

```python
# database/models.py
class AuditLog(Base):
    __tablename__ = 'audit_log'
    id          = Column(Integer, primary_key=True)
    user_id     = Column(Integer, ForeignKey('users.id'), nullable=True)  # nullable für System-Actions
    org_id      = Column(Integer, ForeignKey('organisations.id'), nullable=True)
    action      = Column(String(100), nullable=False)  # 'login', 'session_start', 'session_end', 'profile_update', ...
    target_type = Column(String(100))
    target_id   = Column(Integer)
    details     = Column(Text)  # JSON
    ip_address  = Column(String(64))
    user_agent  = Column(String(500))
    created_at  = Column(DateTime, default=utcnow, nullable=False)
```

**SQLite-Trigger (in `app.py` Init nach `init_db`):**
```python
from sqlalchemy import event, DDL
trg_no_update = DDL("""
CREATE TRIGGER IF NOT EXISTS audit_log_no_update
BEFORE UPDATE ON audit_log
BEGIN
  SELECT RAISE(ABORT, 'audit_log is immutable');
END;
""")
trg_no_delete = DDL("""
CREATE TRIGGER IF NOT EXISTS audit_log_no_delete
BEFORE DELETE ON audit_log
BEGIN
  SELECT RAISE(ABORT, 'audit_log is immutable');
END;
""")
# Execute once on startup
```

`[VERIFIED: SQLite docs sqlite.org/lang_createtrigger.html]` — `RAISE(ABORT, ...)` in BEFORE-Trigger ist der idiomatische Weg für Constraint-Enforcement in SQLite.

**Helper:**
```python
# services/audit.py
def log_action(db, user_id, org_id, action, target_type=None, target_id=None, details=None, request=None):
    entry = AuditLog(
        user_id=user_id, org_id=org_id, action=action,
        target_type=target_type, target_id=target_id,
        details=json.dumps(details) if details else None,
        ip_address=request.remote_addr if request else None,
        user_agent=(request.headers.get('User-Agent') or '')[:500] if request else None,
    )
    db.add(entry)
    db.commit()
```

Wire-up Punkte: `_login_user` (auth.py + oauth.py), `api_start_session`, `api_beenden`, Profil-Update-Routes, Admin-Actions.

### Pattern 3: Flask-Admin Setup mit Auth-Gate

```python
# app.py
from flask_admin import Admin, AdminIndexView, expose
from flask_admin.contrib.sqla import ModelView
from flask import g, abort, redirect, url_for

class SecureIndexView(AdminIndexView):
    def is_accessible(self):
        return getattr(g, 'user', None) and g.user.is_superadmin
    def inaccessible_callback(self, name, **kwargs):
        return redirect(url_for('auth.login'))

class SecureModelView(ModelView):
    def is_accessible(self):
        return getattr(g, 'user', None) and g.user.is_superadmin
    def inaccessible_callback(self, name, **kwargs):
        abort(403)

admin = Admin(app, name='NERVE Admin', template_mode='bootstrap4',
              index_view=SecureIndexView(url='/admin'))

# Dann ModelViews registrieren:
from database.db import SessionLocal
admin.add_view(SecureModelView(User, SessionLocal()))
admin.add_view(SecureModelView(Organisation, SessionLocal()))
admin.add_view(SecureModelView(Feedback, SessionLocal()))
admin.add_view(SecureModelView(AuditLog, SessionLocal(), category='Logs'))
```

`[CITED: flask-admin.readthedocs.io/en/latest/introduction/#authorization-user-permissions]`

⚠️ **SessionLocal() in add_view ist sub-optimal** — Flask-Admin erwartet normalerweise ein `db.session` Scoped-Session-Objekt. Bei bestehendem Muster (manuelle SessionLocal + try/finally) muss ein `scoped_session` für Flask-Admin erstellt werden oder ein Wrapper. PLAN-01 muss das lösen (Option: `flask_sqlalchemy` NICHT einführen, sondern scoped_session manuell in `db.py` ergänzen).

### Anti-Patterns to Avoid

- **Transkript in `audit_log.details` speichern** — verletzt DSGVO-Policy CLAUDE.md (kein Audio/Transkript-Storage). Nur Aggregate.
- **Screenshot-Upload ohne MIME-Check** — User könnte `.exe` als `.png` umbenennen. Content-Type + Extension-Whitelist prüfen.
- **Flask-Admin Auto-Registration aller Models** — würde sensible Tabellen (Stripe, `Session`-Tokens) exposed. Nur explizit registrieren.
- **`FeedbackEvent` umbauen** — bricht Post-Session-Rating-Flow. Neue Tabelle anlegen.
- **Admin-Bereich als Subpage in User-App mounten** — D-05 verbietet explizit. `/admin` als separate Route.
- **Resend-API-Call synchron im Request-Handler** — bei Resend-Downtime hängt User-Request. Empfehlung: best-effort try/except, Email-Failure blockiert nicht den Feedback-Save. (Kein Background-Worker in MVP, nur try/except.)

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---|---|---|---|
| Admin CRUD-Views | Eigenes Jinja-Admin mit Formularen | Flask-Admin ModelView | 95% weniger Code, gratis Pagination/Filter/Search |
| Email-Sending | `smtplib` direkt | `resend` SDK | Bounce-Handling, Template-Management, DKIM, Delivery-Logs gratis |
| Upload-Validation | Eigener MIME-Sniffer | Extension-Whitelist + Werkzeug `secure_filename` + `MAX_CONTENT_LENGTH` | Flask-eingebaut, ausreichend für MVP |
| Immutable-Constraint | Python-only Check | SQLite `BEFORE UPDATE/DELETE` Trigger + App-Check | DB-Level = Defense-in-Depth |
| Password-Reset-Tokens | Eigene Token-Tabelle | `itsdangerous.URLSafeTimedSerializer` (Flask-Core-Dep) | Signed, zeitlich begrenzt, stateless, kein DB-Storage nötig |

**Key insight:** Diese Phase ist Infrastructure-Plumbing. Jede Minute in Custom-Code ist verschwendete Time-to-Market. Flask-Admin + Resend decken 80% der Arbeit ab, der Rest ist Glue.

## Common Pitfalls

### Pitfall 1: "Session" ist ambiguous im Codebase
**Was geht schief:** Planer/Entwickler schreibt `from database.models import Session` — denkt er bekommt Sales-Session, bekommt aber Auth-Token-Tabelle.
**Root Cause:** `database.models.Session` ist die Auth-Token-Tabelle aus altem Flask-Session-System. Die Sales-Session lebt in `ConversationLog`.
**Vermeidung:** PLAN-06 und jeder Code-Kommentar muss explizit `ConversationLog` sagen, nie "Session". Wenn ein Alias helfen würde: `SalesSession = ConversationLog` im relevanten Modul.
**Warnzeichen:** Import-Fehler, oder leere Datensätze beim KPI-Dashboard (weil auf Auth-Token-Tabelle gequeryt).

### Pitfall 2: `avg_deal_wert` vs `avg_deal_value`
**Was geht schief:** CONTEXT D-08 sagt "`avg_deal_value` in `user_profiles`" — existiert aber schon als `users.avg_deal_wert` (Deutsch, falsche Tabelle im Briefing).
**Vermeidung:** Kein neues Feld. Plan 03 dokumentiert: "Nutze bestehendes `users.avg_deal_wert` (aus Phase 04.6), kein neues Feld".
**Warnzeichen:** Zwei Spalten mit ähnlichem Namen in DB.

### Pitfall 3: Flask-Admin + manuelles SessionLocal-Pattern
**Was geht schief:** Flask-Admin erwartet ein Session-Objekt (scoped), bestehender Codebase nutzt `get_session()` mit try/finally. Direkt `SessionLocal()` in `add_view()` leakt Connections und hat Stale-Data-Probleme.
**Vermeidung:** `scoped_session` ergänzen in `database/db.py`, an Flask-Admin übergeben. Alternativ `flask-sqlalchemy` einführen — aber das ist ein grösserer Refactor, besser vermeiden für MVP.
**Warnzeichen:** "Session already closed" Errors, oder veraltete Daten in Admin-Views nach Writes in User-App.

### Pitfall 4: Resend EU-Region vs Default-Region
**Was geht schief:** Resend-SDK default ist US-Region. DSGVO verlangt EU.
**Vermeidung:** Bei Initialisierung explizit EU setzen. `[ASSUMED]` Resend SDK setzt Region per Endpoint/Config — **IN PLAN 05 VERIFIZIEREN** via Resend-Docs oder Test-Call. Alternativ: Resend-Dashboard sagt "EU Data Region" auf Account-Ebene, nicht per Call. Wenn letzteres, muss André das im Resend-Dashboard sicherstellen (CONTEXT sagt DNS ist gesetzt, aber nicht explizit Region).
**Warnzeichen:** `[ASSUMED]` — bei Audit würde Emails via US-Server laufen → DSGVO-Breach.

### Pitfall 5: Screenshot-Upload + `MAX_CONTENT_LENGTH`
**Was geht schief:** `MAX_CONTENT_LENGTH` gilt global. Setzt man es auf 5 MB, sind auch andere POST-Requests (z.B. Audio-Chunks im Live-Assistenten via Socket.IO) limitiert.
**Vermeidung:** Audio geht über Socket.IO, nicht über HTTP POST → MAX_CONTENT_LENGTH betrifft es nicht. Trotzdem: Feedback-Endpoint eigenen Check machen (request.content_length), nicht global erhöhen. Empfehlung: `MAX_CONTENT_LENGTH = 5 * 1024 * 1024` als safe default, explizit im Feedback-Handler Content-Type `multipart/form-data` prüfen.
**Warnzeichen:** Andere Upload-Features brechen unerwartet.

### Pitfall 6: Flask-Admin View-Reihenfolge vor Blueprint-Register
**Was geht schief:** `Admin(app, ...)` muss nach `app = Flask(...)` aber in einer Reihenfolge die Blueprint-Auth-Context respektiert. Bei falscher Reihenfolge läuft `is_accessible()` mit leerem `g`.
**Vermeidung:** Admin nach allen Blueprints registrieren, `before_request` Hook muss vorher laufen. Der bestehende `before_request` in `routes/auth.py` setzt `g.user` — funktioniert auch für `/admin` Routes (Flask-Admin-Views sind normale Flask-Routes).
**Warnzeichen:** `/admin` gibt 500 mit "g has no attribute 'user'".

### Pitfall 7: Audit-Log Trigger blockiert ALTER TABLE
**Was geht schief:** SQLite `BEFORE UPDATE` Trigger blockiert auch `UPDATE` via future Migrations.
**Vermeidung:** Migrationen auf `audit_log` schemaseitig nur ADD COLUMN (append-only), keine UPDATE-Migrationen. Für initial Schema-Setup: Trigger erst CREATE nachdem Tabelle initial angelegt. Bestehender init_db lief ohne Trigger, dann Trigger-CREATE danach.
**Warnzeichen:** Migration-Fehler "audit_log is immutable" beim Deploy.

### Password-Reset Token mit `itsdangerous`
```python
# services/password_reset.py
from itsdangerous import URLSafeTimedSerializer, BadSignature, SignatureExpired
import os

_serializer = URLSafeTimedSerializer(os.environ['SECRET_KEY'], salt='password-reset')

def make_reset_token(user_id: int) -> str:
    return _serializer.dumps(user_id)

def verify_reset_token(token: str, max_age_s: int = 3600) -> int | None:
    try:
        return _serializer.loads(token, max_age=max_age_s)
    except (BadSignature, SignatureExpired):
        return None
```

### Feedback-Upload-Endpoint
```python
# routes/feedback.py
import os, uuid
from flask import Blueprint, request, jsonify, g, abort, send_from_directory, current_app
from services.auth_decorators import login_required, superadmin_required

feedback_bp = Blueprint('feedback', __name__)

ALLOWED_EXTS = {'png', 'jpg', 'jpeg', 'webp'}
ALLOWED_MIMES = {'image/png', 'image/jpeg', 'image/webp'}

@feedback_bp.route('/api/feedback', methods=['POST'])
@login_required
def create_feedback():
    typ = request.form.get('type')  # bug/feature/ux/other
    text = request.form.get('text', '').strip()
    if not typ or not text:
        return jsonify({'error': 'type and text required'}), 400

    screenshot_rel = None
    file = request.files.get('screenshot')
    if file and file.filename:
        ext = file.filename.rsplit('.', 1)[-1].lower() if '.' in file.filename else ''
        if ext not in ALLOWED_EXTS or file.mimetype not in ALLOWED_MIMES:
            return jsonify({'error': 'invalid image'}), 400
        fname = f"{uuid.uuid4().hex}.{ext}"
        upload_dir = current_app.config['FEEDBACK_UPLOAD_DIR']
        os.makedirs(upload_dir, exist_ok=True)
        file.save(os.path.join(upload_dir, fname))
        screenshot_rel = f"feedback/{fname}"

    from database.db import get_session
    from database.models import Feedback
    db = get_session()
    try:
        fb = Feedback(user_id=g.user.id, org_id=g.user.org_id,
                      type=typ, text=text, screenshot_path=screenshot_rel,
                      status='open')
        db.add(fb)
        db.commit()
        fb_id = fb.id
    finally:
        db.close()
    return jsonify({'ok': True, 'id': fb_id})

@feedback_bp.route('/admin/feedback/screenshot/<path:fname>')
@superadmin_required
def serve_screenshot(fname):
    return send_from_directory(current_app.config['FEEDBACK_UPLOAD_DIR'], fname)
```

### Known Threat Patterns for Flask + SQLite Stack

| Pattern | STRIDE | Mitigation |
|---|---|---|
| SQL Injection | Tampering | SQLAlchemy ORM / parametrized queries (already standard in codebase) |
| Privilege Escalation via `/admin` | Elevation | `@superadmin_required` + Flask-Admin `is_accessible` double-gate |
| Screenshot = Malware Upload | Tampering | MIME/Ext whitelist, no execution, served only via gated route |
| Directory Traversal via Screenshot filename | Tampering | UUID-generated filenames, `send_from_directory` (not `open()`) |
| Path in `screenshot_path` column = injection | Tampering | Stored as relative path, joined serverseitig mit hardcoded base |
| IDOR auf Feedback (user sieht fremde Tickets) | Info disclosure | `/admin/*` = superadmin only; `/api/feedback` nur POST, keine GET/LIST in User-App |
| Audit-Log Tampering | Repudiation | SQLite BEFORE UPDATE/DELETE Trigger + App-Layer Reject |
| Email Header Injection | Tampering | Resend SDK handled (keine rohen SMTP-Headers); User-Input nie in From/To/Subject sondern nur HTML-Body escaped |
| Password-Reset-Token Replay | Spoofing | `itsdangerous` TimedSerializer mit `max_age=3600` |
| CSRF auf `/api/feedback` | Tampering | Flask-Session-Cookie SameSite=Lax (bestehend), POST-only |
