# Phase 4.2: Cold Call und Meeting Modi — Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Introduce two distinct live session modes to `/live`:

- **Cold Call:** Consultant audio only. Deepgram single-speaker (no diarization). Passive Claude analysis on Berater transcripts + EWB buttons as manual objection triggers. No customer transcripts processed or stored — DSGVO berechtigtes Interesse (Art. 6 Abs. 1 lit. f).
- **Meeting:** Full diarized analysis (both speakers). Consent pop-up before session start. Accepted → full analysis. Rejected → silent fallback to Cold Call mode.

Training mode is unchanged (AGB/Vertragserfüllung covers it — D-23 from Phase 4).

This phase delivers the mode switching infrastructure, EWB button flow, consent pop-up, and per-session mode tracking. DSGVO legal texts for these modes were already drafted in Phase 4 (D-22, D-26) — no new legal work needed here.

</domain>

<decisions>
## Implementation Decisions

### Mode Selection

- **D-01:** Mode selection via **pre-session overlay on `/live`**. The overlay appears immediately when the user navigates to `/live`, before any Deepgram connection is established. User picks Cold Call or Meeting. No mode switching mid-call.
- **D-02:** **Nice-to-have (v1.2, not blocking):** Persist last-selected mode in `localStorage` so the user doesn't have to re-select each time. Not required for v1 of this phase.
- **D-03:** Active mode displayed as **small badge in the `/live` header** (e.g., "Cold Call" or "Meeting") next to the timer/start button — always visible during the session.

### Meeting Mode — Consent Pop-up

- **D-04:** Consent pop-up appears **directly after Meeting selection** in the mode overlay — before the Start button becomes active. Flow: select Meeting → consent modal shows → Stattgegeben/Abgelehnt → user clicks Start.
- **D-05:** Consent modal is a **standard modal** (not fullscreen). Contains a **hardcoded** Vorleseskript-Satz the user reads aloud to the customer. Not configurable per org in v1 (can become a Pro feature later).
- **D-06:** **"Stattgegeben"** → session starts with full Deepgram diarization (current behavior). Nahtlos, no visual break.
- **D-07:** **"Abgelehnt"** → silently falls back to Cold Call mode. No visible UI break. Session starts exactly as if Cold Call had been selected. The mode badge shows "Cold Call".

### Cold Call — Deepgram Configuration

- **D-08:** Cold Call uses **Deepgram single-speaker mode** (`diarize=false`, `multichannel=false`). Rationale: customer audio must not be processed under DSGVO (berechtigtes Interesse covers only Berater). No diarization cost. Cleaner DSGVO posture — customer data is never sent to Deepgram at all.
- **D-09:** The `start_live_session` Socket.IO event must carry the session mode so `register_audio_handlers` in `deepgram_service.py` can configure the Deepgram `LiveOptions` accordingly (diarize on/off).
- **D-10:** Meeting mode uses Deepgram with full diarization (`diarize=true`) — same as current behavior in `deepgram_service.py`.

### EWB Buttons

- **D-11:** EWB buttons rendered from the **active profile's `einwaende` list**. If the profile has no objections defined: fall back to **DACH-standard list** — "Keine Zeit", "Zu teuer", "Kein Bedarf", "Falscher Zeitpunkt", "Lieferant zufrieden", "Kein Budget", "Muss Chef fragen".
- **D-12:** EWB button click → **immediate Claude Haiku request** with: the specific objection type (button label) + last ~3 transcript segments as context + active profile data (Gegenargumente, Produkt, KI-Anweisung). Result rendered in the AI panel. Same pattern as existing `api_frage` endpoint — new dedicated endpoint `/api/ewb_trigger` or reuse `api_frage` with `typ='ewb'` (planner decides).
- **D-13:** EWB buttons are shown in **Cold Call mode** as the primary manual input. They also appear in **Meeting mode** as a supplement (user can manually trigger even if passive analysis is running).

### Cold Call — Analysis Behavior

- **D-14:** **Passive Claude analysis still runs** in Cold Call mode — Berater transcripts are analyzed automatically (same `_flush_segment` → Claude loop). EWB button is an **additional manual trigger** on top. No change to the existing analysis loop logic.
- **D-15:** In Cold Call, only speaker 0 (Berater) transcripts are processed. Since diarization is off, all transcripts are treated as Berater. No speaker-label filtering needed at the analysis layer.

### Post-Call & Data Model

- **D-16:** `session_mode` column added to `ConversationLog` (values: `'cold_call'`, `'meeting'`). Written in `api_beenden`. Used in post-call analytics and dashboard history. Migration via existing `app.py` `_migrate()` pattern.
- **D-17:** In Cold Call, `ConversationLog` stores: session duration, profile, phase coverage, Berater-side transcripts, EWB button presses (as metadata). No customer transcripts. No speaker 1 data.
- **D-18:** EWB button presses logged in `quick_action_log` (existing list in `live_session.py`) with `typ='ewb'` and the objection type. Preserved in `api_beenden` via existing `qa_count` mechanism — no new log structure needed.

### Claude's Discretion

- Exact consent modal copy (Vorleseskript-Satz) — legally sound German standard sentence
- Visual design of mode selector overlay (card layout vs. button pair)
- Badge styling (color, position, exact label format)
- Whether EWB flow reuses `api_frage` or gets its own `/api/ewb_trigger` endpoint

</decisions>

## Specific Ideas

- Design inspiration: Getclose.ai UI (screenshot in vault) — clean mode indicator and EWB button layout. DSGVO-compliant variant is our USP.
- Consent Vorleseskript: something like *"Ich nutze eine KI-Assistenz, die unser Gespräch in Echtzeit analysiert. Sind Sie damit einverstanden?"* — exact copy Claude's discretion, must be legally sound.
- EWB DACH fallback list: "Keine Zeit", "Zu teuer", "Kein Bedarf", "Falscher Zeitpunkt", "Lieferant zufrieden", "Kein Budget", "Muss Chef fragen" — 7 standard DACH B2B objections.
- Mode badge: minimal, e.g. a pill next to the session timer — "🔒 Cold Call" / "🤝 Meeting" (icon + label, Claude's discretion).

</specifics>
