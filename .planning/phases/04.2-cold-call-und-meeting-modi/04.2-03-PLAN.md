```python
def _get_speaker(result):
    try:
        words = result.channel.alternatives[0].words
        if not words:
            return None
        counts = {}
        for w in words:
            sp = getattr(w, 'speaker', None)
            if sp is not None:
                counts[sp] = counts.get(sp, 0) + 1
        return max(counts, key=counts.get) if counts else None
    except Exception:
        return None
```

Investigate and fix the `LiveOptions` in `_open_deepgram_connection` so Deepgram speaker diarization works reliably on single-channel audio (two speakers on one mic).

**Root cause analysis before editing:**

The Deepgram `nova-2` model supports speaker diarization on single-channel audio via `diarize=True`. The current config is:
- `model="nova-2"` — correct, nova-2 supports diarization
- `diarize=(mode == 'meeting')` — correct conditional
- `smart_format=True` — this CAN suppress per-word speaker attributes in some SDK versions because smart_format post-processes the response and may strip word-level metadata
- No explicit `multichannel` setting — defaults to False, which is correct for single-channel diarization

**The fix:**

1. When `mode == 'meeting'`, set `smart_format=False` — diarization requires raw word objects with `speaker` attribute intact. Smart format reformatting can strip word-level speaker metadata.

2. Add `utterance_end_ms="1000"` when `mode == 'meeting'` — helps Deepgram segment utterances between speakers more reliably.

3. Keep `model="nova-2"` — this is correct. Do NOT change to `nova-2-meeting` which is a separate model for pre-recorded audio, not streaming.

**Concrete change to `_open_deepgram_connection`:**

```python
def _open_deepgram_connection(sid, mode='meeting'):
    client = DeepgramClient(DEEPGRAM_API_KEY)
    connection = client.listen.websocket.v("1")
    connection.on(LiveTranscriptionEvents.Transcript, _make_on_message(sid))
    connection.on(LiveTranscriptionEvents.Open, _make_on_open(sid))
    connection.on(LiveTranscriptionEvents.Error, _make_on_error(sid))
    is_meeting = (mode == 'meeting')
    options = LiveOptions(
        model="nova-2",
        language="de",
        smart_format=not is_meeting,   # disable smart_format in meeting mode — preserves word-level speaker attributes
        interim_results=True,
        endpointing=900,
        punctuate=True,
        diarize=is_meeting,
        utterance_end_ms="1000" if is_meeting else None,
        encoding="linear16",
        sample_rate=SAMPLE_RATE,
    )
    connection.start(options)
    with _sessions_lock:
        _deepgram_sessions[sid] = connection
        _session_modes[sid] = mode
    print(f"[DG] Session gestartet (sid={sid}, mode={mode}, diarize={is_meeting})")
```

**Note on `utterance_end_ms`:** The `LiveOptions` dataclass accepts keyword arguments forwarded to the Deepgram API. If `utterance_end_ms=None` causes an SDK error (some versions don't accept None), use conditional construction instead:

```python
    options_kwargs = dict(
        model="nova-2",
        language="de",
        smart_format=not is_meeting,
        interim_results=True,
        endpointing=900,
        punctuate=True,
        diarize=is_meeting,
        encoding="linear16",
        sample_rate=SAMPLE_RATE,
    )
    if is_meeting:
        options_kwargs['utterance_end_ms'] = "1000"
    options = LiveOptions(**options_kwargs)
```

Use the conditional `options_kwargs` approach — it is safe across all SDK versions and avoids passing None for optional parameters.
