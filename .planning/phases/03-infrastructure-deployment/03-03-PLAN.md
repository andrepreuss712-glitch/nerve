<task type="manual">
  <name>Task 1: Provision Hetzner CX22 VPS and point DNS</name>
  <action>
1. Log in to https://console.hetzner.cloud
2. Create a new server:
   - Location: Nuremberg or Falkenstein (Germany — DSGVO requirement)
   - OS: Ubuntu 22.04 LTS
   - Type: CX22 (2 vCPU, 4 GB RAM, ~4 EUR/month)
   - SSH key: add your existing public key (or create a new key pair first with `ssh-keygen -t ed25519`)
   - Name: nerve-prod
3. Note the assigned IPv4 address (e.g. 49.12.x.x).
4. Log in to your domain registrar for nerve.app.
5. Set two DNS A records:
   - `nerve.app` → your VPS IPv4 address
   - `www.nerve.app` → your VPS IPv4 address
6. Wait for DNS propagation (typically 5-30 minutes). Verify with: `nslookup nerve.app`
  </action>
  <verify>
    <manual>
- `ssh root@YOUR_VPS_IP` connects without a password prompt (SSH key auth working)
- `nslookup nerve.app` returns the VPS IP you just set
    </manual>
  </verify>
  <done>VPS is provisioned, SSH accessible, and DNS records point to the VPS IP</done>

<task type="manual">
  <name>Task 2: Install system dependencies on VPS</name>
  <action>
SSH to the VPS and run these commands. Each block is a logical step — run them in order.

```bash
ssh root@YOUR_VPS_IP
```

**Update system and install runtime dependencies:**
```bash
apt update && apt upgrade -y
apt install -y python3.11 python3.11-venv python3-pip git nginx certbot python3-certbot-nginx ufw
```

**Configure firewall:**
```bash
ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw --force enable
ufw status
```

**Create app user and directory structure:**
```bash
useradd -r -s /bin/false www-data 2>/dev/null || true   # already exists on Ubuntu
mkdir -p /opt/nerve/app
mkdir -p /opt/nerve/venv
mkdir -p /etc/nerve
```

**Clone the repo:**
```bash
cd /opt/nerve/app
git clone https://github.com/YOUR_GITHUB_USER/YOUR_REPO_NAME .
# If repo is private, set up SSH key or use HTTPS with token
```

**Create the virtualenv and install dependencies:**
```bash
python3.11 -m venv /opt/nerve/venv
/opt/nerve/venv/bin/pip install --upgrade pip
/opt/nerve/venv/bin/pip install -r /opt/nerve/app/requirements.txt
/opt/nerve/venv/bin/pip install gunicorn
```

**Set ownership:**
```bash
chown -R www-data:www-data /opt/nerve/app
chown -R www-data:www-data /opt/nerve/venv
```
  </action>
  <verify>
    <manual>
- `/opt/nerve/venv/bin/python --version` outputs Python 3.11.x
- `/opt/nerve/venv/bin/pip show flask` shows Flask version
- `ls /opt/nerve/app/app.py` exists
- `ufw status` shows OpenSSH and Nginx Full ALLOW rules
    </manual>
  </verify>
  <done>Python virtualenv created, NERVE repo cloned, all server dependencies installed</done>
</task>

<task type="manual">
  <name>Task 3: Create /etc/nerve/.env with production secrets</name>
  <action>
Create the production environment file. This file is NOT in the repo — it contains secrets.

```bash
nano /etc/nerve/.env
```

Paste and fill in all values:
```env
# NERVE Production Environment
# Location: /etc/nerve/.env
# Permissions: 600 (readable only by root/service)

SECRET_KEY=<generate a strong random key — see below>
DATABASE_URL=sqlite:////opt/nerve/app/database/nerve.db
DEEPGRAM_API_KEY=<your Deepgram API key>
ANTHROPIC_API_KEY=<your Anthropic API key>
ELEVENLABS_API_KEY=<your ElevenLabs API key>
CORS_ORIGIN=https://nerve.app
MAX_SESSION_HOURS=8
```

**Generate a strong SECRET_KEY** (run this locally or on VPS, copy the output):
```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

**Secure the file:**
```bash
chmod 600 /etc/nerve/.env
chown root:root /etc/nerve/.env
```

**Verify the fail-fast check will pass** (SECRET_KEY must NOT be 'dev-secret-change-me'):
```bash
grep "SECRET_KEY" /etc/nerve/.env
```
  </action>
  <verify>
    <manual>
- `ls -la /etc/nerve/.env` shows permissions `-rw-------` (600)
- `grep SECRET_KEY /etc/nerve/.env` shows a long random hex value (not 'dev-secret-change-me')
- `grep CORS_ORIGIN /etc/nerve/.env` shows `CORS_ORIGIN=https://nerve.app`
    </manual>
  </verify>
  <done>/etc/nerve/.env created with all production secrets, secured to root-only read</done>
</task>

<task type="manual">
  <name>Task 4: Install nginx config and obtain SSL certificate</name>
  <action>
**Copy the nginx config from the repo:**
```bash
cp /opt/nerve/app/deploy/nginx.conf /etc/nginx/sites-available/nerve
ln -s /etc/nginx/sites-available/nerve /etc/nginx/sites-enabled/nerve
rm -f /etc/nginx/sites-enabled/default   # remove the default placeholder
```

**Test and reload nginx (HTTP only first — certbot needs port 80):**
```bash
nginx -t
systemctl reload nginx
```

**Obtain the Let's Encrypt certificate via certbot:**
```bash
certbot --nginx -d nerve.app -d www.nerve.app
```

Certbot will:
1. Ask for your email address (for renewal notices)
2. Ask to agree to terms of service
3. Ask if you want to redirect HTTP to HTTPS — answer Yes (2)
4. Automatically patch the nginx config with SSL cert paths
5. Test the config and reload nginx

**After certbot runs, verify nginx config is valid:**
```bash
nginx -t && systemctl reload nginx
```

**Verify certbot auto-renewal is set up:**
```bash
systemctl status certbot.timer
```
  </action>
  <verify>
    <manual>
- `nginx -t` outputs "syntax is ok" and "test is successful"
- `curl -I https://nerve.app` returns HTTP 200 (may show nginx 502 if gunicorn not started yet — that is OK at this step, we need the SSL handshake to work)
- `curl -I http://nerve.app` returns HTTP 301 redirect to https://
- `ls /etc/letsencrypt/live/nerve.app/` shows fullchain.pem and privkey.pem
    </manual>
  </verify>
  <done>nginx configured with SSL, HTTP redirects to HTTPS, Let's Encrypt cert active and auto-renewing</done>
</task>

<task type="manual">
  <name>Task 5: Install and start the nerve systemd service</name>
  <action>
**Copy the systemd unit file from the repo:**
```bash
cp /opt/nerve/app/deploy/nerve.service /etc/systemd/system/nerve.service
```

**Reload systemd and enable the service:**
```bash
systemctl daemon-reload
systemctl enable nerve
```

**Start the service:**
```bash
systemctl start nerve
```

**Check status immediately:**
```bash
systemctl status nerve --no-pager -l
```

If the service fails to start, check logs:
```bash
journalctl -u nerve -n 50 --no-pager
```

Common startup failures:
- `RuntimeError: SECRET_KEY is insecure` → /etc/nerve/.env has wrong SECRET_KEY value. Fix it, then: `systemctl restart nerve`
- `ModuleNotFoundError` → a dependency is missing from virtualenv. Run: `/opt/nerve/venv/bin/pip install -r /opt/nerve/app/requirements.txt` then restart
- `Address already in use` → another process is on port 8000. Run: `lsof -i :8000` and kill the conflicting process

<task type="manual">
  <name>Task 6: End-to-end verification — HTTPS + WebSocket</name>
  <action>
Final verification that all requirements are met. Run these checks from your local machine and browser.

**Terminal checks (from local machine):**
```bash
# HTTPS responds with 200
curl -I https://nerve.app

# HTTP redirects to HTTPS
curl -I http://nerve.app

# TLS certificate is valid and issued for nerve.app
echo | openssl s_client -connect nerve.app:443 -servername nerve.app 2>/dev/null | openssl x509 -noout -dates -subject
```

**Browser verification:**
1. Open https://nerve.app in Chrome/Firefox
2. Open DevTools → Network tab
3. Filter by "WS" or "websocket"
4. Reload the page
5. Click on the Socket.IO connection entry
6. Confirm the Status column shows `101 Switching Protocols`
7. If you see repeated HTTP 200 requests to `/socket.io/?EIO=4&transport=polling` — WebSocket upgrade is failing. Check nginx headers in `deploy/nginx.conf` are present in the active `/etc/nginx/sites-enabled/nerve`.

**VPS health check:**
```bash
ssh root@YOUR_VPS_IP "systemctl status nerve --no-pager && nginx -t"
```
