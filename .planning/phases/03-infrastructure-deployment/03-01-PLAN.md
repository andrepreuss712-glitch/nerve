  <name>Task 3: Enable SQLite WAL mode in database/db.py</name>

  <action>
Per D-06 (INFRA-04): SQLite's default journal mode is DELETE (rollback journal). Under multi-threaded Flask-SocketIO with concurrent reads and writes, this causes lock contention. WAL (Write-Ahead Logging) allows concurrent reads alongside writes.

Add the SQLAlchemy event listener pattern immediately after the `engine = create_engine(...)` line in database/db.py:

```python
from sqlalchemy import event

@event.listens_for(engine, 'connect')
def set_wal_mode(dbapi_connection, connection_record):
    cursor = dbapi_connection.cursor()
    cursor.execute('PRAGMA journal_mode=WAL')
    cursor.close()
```

The `from sqlalchemy import event` import should be added to the existing sqlalchemy import line at the top of the file. Change:
```python
from sqlalchemy import create_engine
```
to:
```python
from sqlalchemy import create_engine, event
```

Then add the `@event.listens_for` block right after the `engine = create_engine(...)` line (before `SessionLocal = sessionmaker(...)`).

Note: `PRAGMA journal_mode=WAL` only applies to SQLite. The guard `if 'sqlite' in _DATABASE_URL` already wraps `_connect_args` — the WAL listener is safe to apply unconditionally because the pragma is a no-op on non-SQLite engines and we are SQLite-only per config defaults.
