# Phase 04.6.2 Plan 03: OAuth Polish (D-04 + D-06) Summary

Conditional `prompt=consent` für Microsoft (Silent-SSO bei bekanntem Tenant) plus Wording-Angleichung des Consumer-Account-Blocks auf den CONTEXT.md-Text.

## What was built

**D-04 — Consumer-Tenant-Block (Wording-Polish):**
- `routes/oauth.py` Zeilen ~160-165: Bestehender Tenant-ID-Check (`9188040d-...`) bleibt unverändert. Fehlermeldung wurde auf den CONTEXT.md-Wortlaut aktualisiert: *"NERVE benötigt ein Microsoft 365 Business-Konto. Private Microsoft-Konten (Hotmail, Outlook, Live) werden nicht unterstützt. Bitte verwende deinen Firmen-Account oder melde dich mit Google an."*
- `templates/landing.html` errMap (Zeile 794): Identische Mirror-Aktualisierung des `personal_account` Strings — Banner zeigt jetzt denselben Wortlaut.

**D-06 — Conditional prompt=consent:**
- Neuer Helper `_is_known_oauth_tenant(provider, email_hint)` (oauth.py): Extrahiert Domain aus Email-Hint, queryt `User`-Tabelle nach `oauth_provider == provider` AND `lower(email) LIKE '%@<domain>'`. Liefert True wenn mindestens ein User aus diesem Tenant via diesem Provider eingeloggt ist.
- `microsoft_login()`: Liest `?login_hint=` aus Query-Params. Wenn vorhanden + Tenant bekannt → kein `prompt=consent` (Silent-SSO möglich). Wenn vorhanden + Tenant unbekannt → `prompt=consent` (Service-Principal-Provisioning). Wenn kein Hint → konservativer Fallback `prompt=consent` (preserves AADSTS900971-Workaround). Original-Kommentar zur AADSTS900971-Begründung bleibt erhalten.
- `google_login()`: Reicht `login_hint` durch falls vorhanden, kein `prompt=consent` (Google hat kein Service-Principal-Provisioning).
- TODO-Kommentar in beiden `*_login()`-Funktionen: Frontend kann zukünftig `?login_hint=<email>` mitsenden.
