- Bei `microsoft_login()` (vor Redirect zum Provider) wissen wir noch NICHT welcher User sich einloggen will — Email kommt erst aus dem Token nach Callback.
- Lösung: Heuristik per Cookie/Session ODER per `login_hint` Query-Param (falls UI ihn liefert) ODER pragmatisch: `prompt=consent` weglassen by default und nur beim Microsoft `microsoft_failed`-Retry-Path (AADSTS900971-Workaround) gezielt setzen.
- Pragmatische Wahl (siehe Action unten): Default = kein consent. Bei explizitem Query-Param `?force_consent=1` (Admin-Trigger) oder bei einer Cookie-Markierung `nerve_oauth_first_login=1` → consent senden.

Existing Microsoft-Kommentar (Zeilen 142-144):
```
# prompt=consent zwingt Azure dazu, den Service-Principal im Ziel-Tenant beim
# ersten Login neu zu provisionieren. Umgeht einen bekannten AADSTS900971-Bug
# bei Multi-Tenant Apps aus Shell-Home-Tenants.
```
→ Wichtig: Der Original-Grund für prompt=consent ist nicht "User-Consent", sondern Azure-Service-Principal-Provisioning. D.h. das Weglassen ist beim ALLERERSTEN Login eines NEUEN TENANTS riskant. Lösung: prompt=consent senden wenn kein User mit @<domain> in der DB existiert (Tenant-First-Login-Heuristik).

1. Helper-Funktion oben in der Datei einfügen (nach Imports, vor `oauth_bp = Blueprint(...)`):
   ```python
   def _is_known_oauth_tenant(provider, email_hint):
       """Liefert True wenn bereits ein User aus dieser Email-Domain via diesem Provider eingeloggt ist.
       Heuristik für D-06: Tenant ist Azure bekannt → Service-Principal existiert → kein prompt=consent nötig.
       """
       if not email_hint or '@' not in email_hint:
           return False
       domain = email_hint.split('@', 1)[1].strip().lower()
       if not domain:
           return False
       db = get_session()
       try:
           # Suche User mit gleicher Email-Domain UND oauth_provider gesetzt
           from sqlalchemy import func
           existing = db.query(User).filter(
               User.oauth_provider == provider,
               func.lower(User.email).like(f'%@{domain}')
           ).first()
           return existing is not None
       finally:
           db.close()
   ```

2. `microsoft_login()` (Zeile 137) anpassen:
   ```python
   @oauth_bp.route('/auth/microsoft')
   def microsoft_login():
       if not (MICROSOFT_CLIENT_ID and MICROSOFT_CLIENT_SECRET):
           return _oauth_error_redirect('not_configured', 'Microsoft-Login nicht konfiguriert.')
       redirect_uri = url_for('oauth.microsoft_callback', _external=True)
       # D-06: prompt=consent nur beim ersten Login eines Tenants senden.
       # Hintergrund: Original-Grund war Azure-Service-Principal-Provisioning (AADSTS900971-Workaround).
       # Wenn bereits ein User aus dieser Email-Domain via Microsoft eingeloggt ist, ist der
       # Service-Principal im Ziel-Tenant bereits provisioniert → Silent-SSO möglich.
       login_hint = request.args.get('login_hint', '').strip()
       extra = {}
       if login_hint:
           extra['login_hint'] = login_hint
           if not _is_known_oauth_tenant('microsoft', login_hint):
               extra['prompt'] = 'consent'
       else:
           # Kein Hint → konservativ consent senden (alter Default, Tenant-Provisioning sicher)
           extra['prompt'] = 'consent'
       return oauth.microsoft.authorize_redirect(redirect_uri, **extra)
   ```
