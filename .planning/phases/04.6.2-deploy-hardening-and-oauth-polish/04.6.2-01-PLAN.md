Den scp-Block (Zeilen 15-20) durch rsync ersetzen. Anforderungen:

1. Nutze `rsync -avz --delete` über SSH mit dem bestehenden Key `~/.ssh/nerve_vps`.

   - Quelle: `./` (gesamtes Repo-Root)
   - Ziel: `"$VPS_HOST:$APP_DIR/"`

2. Excludes (per `--exclude=PATTERN`, in genau dieser Reihenfolge):
   - `.git`
   - `.gitignore`
   - `.env`
   - `.env.example` NICHT excluden (darf mit)
   - `.planning`
   - `.claude`
   - `__pycache__`
   - `*.pyc`
   - `node_modules`
   - `logs`
   - `salesnerve_log_*.txt`
   - `deploy`
   - `deploy.sh`
   - `*.db-journal`
   - `*.db-wal`
   - `*.db-shm`

3. CRITICAL — Prod-DB schützen (D-01 locked decision):
   - Exclude: `database/*.db`
   - Begründung-Kommentar im Skript: `# Prod-SQLite NICHT überschreiben — Schema-Code (database/models.py, db.py) wird übertragen, .db-Dateien nicht.`
   - Zusätzlich `--exclude=database/salesnerve.db` als Defense-in-Depth (explizit, falls Pattern später geändert wird).

4. Vor dem echten rsync einen Dry-Run-Schritt einbauen, der bei Skript-Aufruf mit `--dry-run` als erstes Argument nur `rsync ... --dry-run` ausführt und dann exitet. Default ohne Argument = echter Deploy.
   - Pattern:
     ```bash
     RSYNC_FLAGS="-avz --delete"
     if [[ "${1:-}" == "--dry-run" ]]; then
       RSYNC_FLAGS="$RSYNC_FLAGS --dry-run"
       echo "[deploy] DRY RUN — keine Dateien werden geändert"
     fi
     ```

5. Heredoc-Block (pip install, nginx, systemctl) ab Zeile 22 unverändert lassen, aber im Dry-Run-Modus überspringen (early-exit nach rsync).

6. Echo-Output anpassen: `[deploy] Uploading via rsync (excludes: .git, .env, .planning, *.db, ...)`

Nicht ändern: VPS_HOST, APP_DIR, VENV_DIR, SSH-Key-Pfad, nginx-Config, systemctl-Restart.
