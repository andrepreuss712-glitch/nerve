## Decisions Made

- Loading overlay uses `display:flex` not `display:block` — flex centering is required for the card to appear centered in the full-screen overlay
- `pcLoading` variable is declared before the `try` block so it is accessible in the `catch` block for cleanup — avoids stuck loading state on API errors
- All `body.kompakt .element { display:none }` rules removed — with floating overlay approach there is no need to hide main content, the panel sits above it
- `kompakt-panel` width set to 320px as specified in the plan (PROJECT.md says 380px floating panel, but plan spec uses 320px)
