Investigate and fix the training start flow (BUG-01, BUG-03):

1. **Anruf-Button (BUG-01):** Read `checkReady()` function (approx line 239) — verify it enables `t-callBtn` when both `t-profileSelect` and a scenario are selected. Read `startTraining()` (approx line 468) — verify the fetch POST to `/training/start` sends correct payload. Check `startFreizeichen()` for AudioContext creation — if AudioContext is suspended, add `audioCtx.resume()` before playing the ring tone. The root cause per research is likely: AudioContext auto-suspend policy. Fix: In `startFreizeichen()`, before `oscillator.start()`, add:
```javascript
if (audioCtx.state === 'suspended') { await audioCtx.resume(); }
```
Make `startFreizeichen` async if needed, or use `.then()`.

2. **TTS playback (BUG-03):** Search for ElevenLabs audio playback code — likely in the `playTTS()` or similar function that receives audio from the server and plays via `new Audio()` or AudioContext. Verify: (a) the audio src URL is correct, (b) the Audio element is not blocked by autoplay policy. If `new Audio(url).play()` is used, wrap in user-gesture context or add error handling:
```javascript
audio.play().catch(e => console.warn('[TTS] Autoplay blocked:', e));
```

3. **PTT mic (BUG-02 training side):** Verify `startRec()` calls `navigator.mediaDevices.getUserMedia({audio:true})`. This should work over HTTPS. Add a console log for debugging:
```javascript
console.log('[MIC] Requesting getUserMedia...');
```
If getUserMedia fails, show a user-visible error in the chat:
```javascript
.catch(err => { addSystemMsg('Mikrofon-Zugriff fehlgeschlagen: ' + err.message); });
```

4. **Live mic (BUG-02 live side):** Per research, the live mic is server-side PyAudio — VPS has no audio hardware. This is a known architectural limitation (INFRA-05 removed PyAudio from server requirements). No code fix possible in this phase. Add a comment at the top of `static/app.js` documenting this:
```javascript
// NOTE: Live mic is server-side PyAudio. On VPS without audio hardware,
// mic capture is non-functional. Browser-based WebRTC mic is Phase N+1 scope.
```
