# Phase 04.2.1 Plan 05: EWB AI Selection Summary

**Claude Haiku ranks top-2 likely objections every 3rd analysis cycle, updating PiP EWB buttons live with gold accent border**

## Accomplishments
- Added `rank_ewb()` to `claude_service.py` — isolated Haiku call with JSON-array response, validates returned types against profile list, falls back to first 2 on any error
- Integrated throttled EWB ranking into `analyse_loop()` (every 3rd cycle), storing result in `ls.state['ewb_top2']` and logging to console
- Exposed `ewb_top2` via `/api/ergebnis` polling endpoint and consumed in frontend polling loop to call `renderPipEwbButtons(aiTop2)`
- Added `pip-ewb-ai-selected` CSS class with gold `#d4a853` border to visually distinguish AI-ranked buttons from fallback profile buttons

## Files Created/Modified
- `services/claude_service.py` - Added `rank_ewb()` function, `_ewb_rank_counter`, throttled ranking call in `analyse_loop()`
- `services/live_session.py` - Added `ewb_top2: None` to state dict, reset in `reset_session()`
- `routes/app_routes.py` - Added `ewb_top2` to `/api/ergebnis` payload
- `static/app.js` - Consume `ewb_top2` in polling loop, pass to `renderPipEwbButtons()`; add `pip-ewb-ai-selected` class to AI-selected buttons in `renderPipEwbButtons()`
- `templates/app.html` - Added `.pip-ewb-ai-selected` and `.pip-ewb-ai-selected:hover` CSS rules

## Decisions Made
- Option B chosen (separate `rank_ewb()` function) over Option A (modifying existing prompt) — the existing objection-detection prompt is critical path; adding EWB ranking to it risks format changes that break `_parse_json()` or Einwand detection accuracy.
- Throttle set to every 3rd cycle as plan specified — reduces Haiku cost while keeping PiP reasonably fresh.
- `rank_ewb()` validates that returned EWB names are in the profile list — prevents hallucinated types from being shown as buttons (which would call `triggerEwb()` with an unknown type, producing a bad API request).
- No EWB ranking when profile has no `einwaende` — fallback handled client-side by `renderPipEwbButtons()` which already falls back to first 2 from `window._profileEinwaende`.
