## Spacing Scale

Declared values (must be multiples of 4). Source: `nerve.css` CSS custom property conventions + existing component padding audit.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, pill tab inner gaps |
| sm | 8px | Between inline elements (mode badge + timer), EWB button row gaps |
| md | 16px | Default element padding, sidebar nav item padding, PiP panel inner padding |
| lg | 24px | Dashboard section padding, glass panel padding |
| xl | 32px | Between major dashboard sections (North Star to Usage row) |
| 2xl | 48px | Page-level vertical margin between grid zones |
| 3xl | 64px | Reserved for future use — not used in this phase |

Exceptions:
- Sidebar width: 204px (existing `.g-sidebar`, must not change — CSS var driven)
- Sidebar collapsed width: 52px (icon-only, new for this phase)
- PiP window start size: 380px wide × 280px tall (Document PiP API initial dimensions)
- App header height: 52px (existing, must not change — other elements depend on this)
- Touch targets on PiP tab buttons: minimum 44px tall

## Component Inventory

All components hand-written. This section defines what exists, what is restructured, and what is new.

### Existing Components to Restructure

#### Sidebar (`base.html`, `.g-sidebar`)
- **Current:** 204px fixed, 8+ nav items always visible, no collapse
- **New contract:**
  - Add collapse toggle button at top of sidebar (icon: `chevrons-left` / `chevrons-right`, Lucide)
  - Collapsed state: 52px wide, icon-only nav items, no text labels
  - Expanded state: 204px, icon + label (unchanged visual)
  - Collapse state persisted in `localStorage('sn_sidebar_collapsed')`
  - Transition: `width 0.2s ease` — no animation on individual items, just width
  - **6 nav items only:** Dashboard | Live-Assistent | Training | Analytics | Profil | Einstellungen
  - Existing items (Coach, Team, Changelog, Methodik, Logs) hidden via `display:none` — not deleted
  - "Analytics" nav item links to existing `/logs` route — label change only, no backend change
  - User block at bottom: retain existing structure, add Plan-Badge chip (e.g. "Pro") rendered from `g.org` plan data, teal badge variant (`--badge-primary-bg` / `--badge-primary-text`)

#### Dashboard (`dashboard.html`)
- **Current:** Greeting block, quote-of-day, 4 SVG stat circles, 2-col grid, heatmap, bottom grid
- **New contract:** See Dashboard Layout section below
- **Removed:** KI-Motivationsspruch / quote-of-day block, 4 SVG stat circles, dominant heatmap, large achievements list
- **Retained:** Usage widget structure (repurposed as new progress bars), recent sessions list, quick actions panel

#### Live Vollbild Header (`app.html`, `.header` + `.profile-bar` + `.phasen-bar`)
- **Current:** 3 stacked horizontal bars = ~104px of chrome before main content
- **New contract:** Merge to single 52px header
  - Retain: logo mark, mode badge, timer, Pause/Resume button, Beenden button
  - Move profile switcher into a dropdown triggered from header — not a persistent bar
  - Phasen-bar: removed from always-visible position. Opener section is now a tab in the AI panel (see D-19), not a chrome bar
  - Compact toggle button moves into header (right side) instead of fixed-position overlay

#### EWB Bar (`app.html`, existing `#ewb-bar` / `#kp-bar`)
- **Current:** All 7 EWB buttons always visible in both full and compact views
- **New contract (Kompakt):** Show 2 KI-selected buttons + "Alle ▾" expand trigger
- **New contract (Vollbild):** Show all available buttons (unchanged, but styled consistently)
- KI selection mechanism: rank `einwaende` from active profile by probability score from last ~5 transcript segments; top 2 shown. Fallback: first 2 from DACH standard list.

### New Components

#### Kompakt PiP Panel (Document Picture-in-Picture)
- **Trigger:** Replaces existing `.btn-view-toggle` + `body.kompakt` floating overlay pattern
- **API:** `window.documentPictureInPicture.requestWindow({ width: 380, height: 280 })`
- **Fallback:** If `documentPictureInPicture` not available (non-Chrome), fall back to existing fixed-position `.kompakt-panel` overlay with a visible warning badge: "PiP nicht unterstützt — Chrome 116+ erforderlich"
- **Structure:**

```
.pip-window
  .pip-header            (height: 36px, flex row, space-between)
    .pip-mode-badge      (mode text: "Cold Call" or "Meeting")
    .pip-timer           (00:42, JetBrains Mono, teal)
    .pip-to-fullscreen   (icon: maximize-2, 24px tap target + "Vollbild" text label inline, 12px weight 400)
  .pip-tabs              (height: 36px, 4 tabs, pill style)
    [Opener] [Einwand] [Coaching] [EWB]
  .pip-content           (flex: 1, min-height: 120px, padding: 16px)
    .pip-tip-text        (14px, weight 400, line-height 1.5)
  .pip-ewb-row           (height: 44px, flex row, gap: 8px, padding: 0 16px)
    .pip-ewb-btn x2      (pill shape, ghost style)
    .pip-ewb-all         ("Alle ▾", ghost, smaller)
```

- **PiP to-fullscreen element:** Renders as icon (`maximize-2`, 16px) followed by text "Vollbild" (12px, weight 400, muted) inline in a flex row. Not icon-only. No tooltip needed.

- **Tab auto-switching behavior:**
  - Default active tab on session start: Opener
  - KI drives tab switching automatically based on analysis results (socket events)
  - If user clicks a tab manually: that tab is "locked" — KI will not override
  - Lock indicator: subtle lock icon (Lucide `lock`, 12px) appears in active tab label when manually locked
  - Lock released when: user clicks locked tab again, or new KI result arrives for a different tab and user has not interacted for >30 seconds (Claude's Discretion on timeout)
  - State variable: `window._pipTabLocked = null | 'opener' | 'einwand' | 'coaching' | 'ewb'`

- **Opener Tab content:**
  - Shows elevator pitch / Einstiegssatz from active profile `phasen[0]` or profile opener field
  - Opener recognition: when Claude Haiku detects opener phase complete (via existing analysis stream), emit `opener_complete` socket event → KI switches to Einwand/Coaching tab
  - If no profile opener data: show placeholder "Kein Opener im Profil hinterlegt — Profil bearbeiten"

#### Dashboard North Star Widget
- Teal number display (28px, JetBrains Mono, weight 600) showing weekly score 0–100
- Trend indicator: arrow-up or arrow-down icon (Lucide, 16px) + percentage delta (14px, weight 400)
- Label: "Score diese Woche" (12px, uppercase, muted)
- Contained in glass panel, top-left of dashboard main area

#### Dashboard Next Best Action Card
- Left border accent: 3px teal (reuse existing nudge-card pattern)
- Icon: `lightbulb` (Lucide, 16px, gold `#d4a853`)
- Heading: 16px, weight 600, problem description
- Body: 14px, weight 400, teal link → target action
- Logic: compare score sub-areas (Opener / Einwand / Closing) from last 3 sessions; surface weakest area. If no sessions: show onboarding prompt.
- Dismiss: `×` button, stores dismissed state in `localStorage('sn_nba_dismissed_{week_key}')`

#### Sidebar Plan Badge
- Inline chip in user block: e.g. "Pro"
- Style: `--badge-primary-bg` + `--badge-primary-text`, 12px, weight 600
- Source: `g.org.plan` (existing org model field)

## Interaction Contract

### State Machine — PiP Tab

```
State: auto-mode (default)
  KI emits result → highlight matching tab, update content

State: locked (user clicked tab)
  Manual tab active, lock icon visible
  KI emits result → content updates but tab does not switch
  Exit locked: user clicks locked tab again OR timeout (30s of no user interaction after last KI result)
```

### Sidebar Collapse

- Toggle button: fixed at top of sidebar, always visible
- Keyboard: no keyboard shortcut required in this phase
- On collapse: text labels fade out, icon stays, tooltip appears on hover (native `title` attribute)
- Expand animation: `width 0.2s ease`

### EWB "Alle ▾" Expand

- Click opens full EWB list as inline expansion below the 2 visible buttons — not a modal
- Expanded state: all profile EWBs visible as pill buttons, same row layout as existing full view
- Close: click "Alle ▾" again (toggles), or clicking an EWB button closes the expansion

### Next Best Action Dismiss

- Dismiss stores `localStorage('sn_nba_dismissed_{YYYY-Www}')` where key includes ISO week
- Card reappears next week if user's scores have changed

### Document PiP Lifecycle

- PiP opens: `documentPictureInPicture.requestWindow({width: 380, height: 280})`
- PiP content: clone/move pip-window div into PiP document body, re-attach Socket.IO listeners
- PiP closed by user (native browser close): emit synthetic `pip_closed` event, restore Vollbild as primary
- `localStorage('sn_kompakt')` still written on PiP open/close for backward compat with existing body.kompakt checks
- PiP window title: "NERVE — Live"
