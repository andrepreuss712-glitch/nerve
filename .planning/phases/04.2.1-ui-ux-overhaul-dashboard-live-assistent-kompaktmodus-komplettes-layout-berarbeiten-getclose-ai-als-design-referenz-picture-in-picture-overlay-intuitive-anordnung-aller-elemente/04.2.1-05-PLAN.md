## Goal
Score and rank the top 2 most likely EWBs (Einwandbehandlungen) from the active profile based on the last ~5 transcript segments using Claude Haiku. Display these 2 pre-selected EWBs in the PiP window's bottom bar and update them as the conversation progresses. Vollbild EWB bar remains unchanged (shows all buttons per D-17).

## Context
- `static/app.js` has `renderEwbButtons()` which renders all EWBs from the active profile to both `#ewbBar` (Vollbild) and `#kp-ewbBar` (old kompakt, now PiP `pip-ewb-row`).
- Plan 03 added `renderPipEwbButtons(aiTop2)` which accepts an optional `aiTop2` array of 2 EWB type names and renders them to the PiP EWB row. If `aiTop2` is null, it falls back to the first 2 from the profile.
- Profile einwaende structure: `{einwaende: [{typ: "Zu teuer", ...}, {typ: "Kein Budget", ...}, ...]}` -- stored in profile `.daten` JSON.
- Backend: `services/claude_service.py` contains `analysiere_mit_claude()` which runs periodic analysis on transcript. This is the hook point for EWB scoring.
- Socket event `ergebnis` already pushes analysis results to the frontend. Extend it to include `ewb_top2` field.
- Claude Haiku is already used for analysis (`claude-haiku-4-5-20251001`). EWB scoring is a lightweight inference on last ~5 transcript segments.
- DACH fallback list: `DACH_FALLBACK_EINWAENDE` in app.js line 136 (7 standard objections).

**Option B (if prompt modification is too risky):**

Create a separate lightweight function `rank_ewb(transcript_segments, einwaende_list)` that makes a separate Haiku call with a focused prompt:
```python
def rank_ewb(transcript_segments, einwaende_list):
    """Rank top 2 most likely upcoming objections based on recent transcript."""
    if not transcript_segments or not einwaende_list:
        return einwaende_list[:2]
    try:
        from config import ANTHROPIC_API_KEY
        import anthropic
        client = anthropic.Anthropic(api_key=ANTHROPIC_API_KEY)
        prompt = f"""Basierend auf diesen letzten Gesprachssegmenten:
{chr(10).join(transcript_segments[-5:])}

Welche 2 dieser Einwande kommen am wahrscheinlichsten als nachstes?
Einwande: {', '.join(einwaende_list)}

Antworte NUR mit einem JSON-Array der 2 wahrscheinlichsten: ["typ1", "typ2"]"""

        msg = client.messages.create(
            model='claude-haiku-4-5-20251001',
            max_tokens=100,
            messages=[{'role': 'user', 'content': prompt}]
        )
        import json
        result = json.loads(msg.content[0].text.strip())
        if isinstance(result, list) and len(result) >= 2:
            return result[:2]
    except Exception:
        pass
    return einwaende_list[:2]
```

Call this every 3-5 analysis cycles (not every cycle -- throttle to reduce API cost). Use a counter:
```python
_ewb_rank_counter = 0
# In the analysis loop:
_ewb_rank_counter += 1
if _ewb_rank_counter % 3 == 0:
    ewb_top2 = rank_ewb(recent_segments, einwaende_list)
```
