# Phase 04.2.1 Plan 03: Document PiP Kompaktmodus Summary

**One-liner:** Document Picture-in-Picture window with 4 tabs (Opener/Einwand/Coaching/EWB), mini-header, timer sync, and fallback overlay for non-Chrome browsers replacing the old .kompakt-panel toggle.

## What Was Built

### Task 1: PiP HTML Structure (templates/app.html)
- Added full PiP CSS block (36 rules: pip-header, pip-tabs, pip-tab-active/locked, pip-content, pip-ewb-row, pip-ewb-btn, pip-ewb-expanded)
- Added `#pip-window` div with mini-header (mode badge + timer + Vollbild link), 4 tab buttons, 4 content panels, EWB row, fallback warning
- Renamed `#kompaktPanel` to `id="kompakt-fallback"` with `display:none` — only shown as fallback
- Rewired `viewToggleBtn` to `onclick="openPipWindow()"` with label "Kompakt"
- Rewired `.kp-btn-expand` to `closePipToVollbild()`
- Added `window._profileData` and updated `window.sessionMode = null` in pre-app.js script block

### Task 2: PiP JavaScript (static/app.js)
- `getPipElement(id)`: finds elements in PiP window first, fallback to main document
- `openPipWindow()`: async, requests 380x280 Document PiP window, copies stylesheets (inline CSS) or links (CORS sheets), moves `#pip-window` into PiP body, loads Lucide in PiP context, handles `pagehide` to restore DOM
- `closePipToVollbild()`: closes PiP window
- Tab state machine: `handlePipTabClick`, `setPipTabFromKI` (30s lock timeout), `activatePipTab`
- `initPipContent()`: sets mode badge, reads profile phasen[0] for opener text, renders EWB buttons, starts timer sync, activates opener tab
- `renderPipEwbButtons(aiTop2)`: top 2 einwaende + "Alle" expand button
- `togglePipEwbExpand()`: creates expanded div via `ownerDocument.createElement()` for correct PiP context
- `syncPipTimer()`: setInterval mirrors `#timer` text into `#pip-timer` every 1s
- `updatePipFromErgebnis(d)`: wired at end of `zeigeKarte()` — updates pip-einwand-text, switches tab via `setPipTabFromKI('einwand')`
- `updatePipFromCoaching(d)`: wired at end of `socket.on('coaching')` — updates pip-coaching-text, switches tab via `setPipTabFromKI('coaching')`

### Task 3: Profile Data for PiP (templates/app.html + static/app.js)
- `window._profileData` set from `active_profile.daten` JSON in template (already had `_profileEinwaende`)
- `window.sessionMode = null` initialized in template before app.js
- `activateSession()` in app.js now sets `window.sessionMode = sessionMode` to sync for PiP badge

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Synced window.sessionMode in activateSession()**
- **Found during:** Task 3
- **Issue:** `initPipContent()` reads `window.sessionMode` for PiP mode badge. The local `let sessionMode` in app.js would not be accessible via `window.sessionMode` — badge would always show fallback text.
- **Fix:** Added `window.sessionMode = sessionMode;` at start of `activateSession()` to sync before PiP content init.
- **Files modified:** static/app.js
- **Commit:** e9ab45f

**2. [Rule 2 - Missing Critical] Used ownerDocument.createElement() for PiP expand div**
- **Found during:** Task 2
- **Issue:** Plan used `document.createElement('div')` for the expanded EWB section, but when PiP is open, elements must be created in the PiP window's document context to render correctly.
- **Fix:** Changed to `expandDiv.ownerDocument.createElement()` pattern using `row.ownerDocument` context.
- **Files modified:** static/app.js
- **Commit:** 17b980e

## Known Stubs

None — all PiP content is wired to live data sources:
- Opener text: reads `window._profileData.phasen[0]` (live profile data)
- EWB buttons: reads `window._profileEinwaende.einwaende` (live profile data)
- Timer: syncs from `#timer` element (live session timer)
- Einwand panel: updated via `updatePipFromErgebnis()` on every AI result
- Coaching panel: updated via `updatePipFromCoaching()` on every coaching socket event
- Mode badge: set in `initPipContent()` from `window.sessionMode`
