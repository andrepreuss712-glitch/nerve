# Phase 04.2.1 — Vollbild Chrome Reduction: 3 Bars to 1 Header + Opener in AI Panel
**Plan:** 04.2.1-04-PLAN.md
**Req IDs:** D-02, D-17, D-18, D-19
**Depends on:** 04.2.1-03 (PiP implementation must be in place since this plan moves the Kompakt toggle button into the new single header)

## Goal
Reduce the `/live` Vollbild chrome from 3 stacked bars (~120px) to a single 52px header. Move profile switcher into a dropdown in the header. Remove Phasen-Bar from always-visible position. Add Opener section as a new section in the AI panel column. Move Kompakt toggle into the header.

## Context
- `templates/app.html` has 3 chrome bars:
  1. `.header` (52px, line ~14) -- logo, KB display, Skript btn, Frage btn, email, Log btn, Beenden, Pause, timer
  2. `.profile-bar` (~36px, line ~62) -- profile label, profile name, view toggle btn, profile switch dropdown
  3. `.phasen-bar` (~36px, appears conditionally) -- phase step buttons
- Target: single 52px header containing: `[N] NERVE | [Mode Badge] [Timer] | [Profile dropdown] | [Pause] [Beenden] | [Kompakt button]`
- Items REMOVED from header: Kaufbereitschaft display (D-14, hidden from live view), Skript toggle, "Frage stellen" button, user email label, "Log" download button.
- Phasen-bar: hidden entirely (display:none, not deleted) per D-15.
- Profile-bar: `.profile-bar` element hidden. Profile switcher merged into header as ghost dropdown button reusing existing `.profile-menu` + `.btn-profile-switch` pattern.
- Opener section (D-19): add as first section in AI panel column (`#ai` scroll container). Shows opener text from active profile `phasen[0]`. Collapses/hides when user scrolls or after manual toggle.
- Vollbild remains as full detail view (D-17): transcript, all EWB buttons, coaching, AI cards all stay.
- Kompakt toggle button moves into header right side, calling `openPipWindow()` from Plan 03.

### 5. Ensure profile menu toggle works in new position

The profile menu toggle function `toggleProfileMenu()` may already exist. If not, add:
```javascript
function toggleProfileMenu() {
  var menu = document.getElementById('profileMenu');
  if (menu) menu.classList.toggle('open');
}
// Close on outside click
document.addEventListener('click', function(e) {
  var menu = document.getElementById('profileMenu');
  var toggle = document.getElementById('profileToggle');
  if (menu && toggle && !toggle.contains(e.target) && !menu.contains(e.target)) {
    menu.classList.remove('open');
  }
});
```

Update the `.profile-menu` CSS to position relative to the new header button (not the old profile-bar):
```css
.header .profile-menu { position: absolute; top: calc(100% + 4px); right: 0; }
```
