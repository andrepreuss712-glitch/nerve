# Phase 04.2.1: UI/UX Overhaul — Dashboard, Live-Assistent, Kompaktmodus — Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Complete layout overhaul of three surfaces:
1. **`/live` Kompaktmodus** — the new PRIMARY mode (not fullscreen). Document Picture-in-Picture API. Single info at a time. 2-second glance UX.
2. **`/live` Vollbild** — demoted to secondary/optional (before/after call, detail view). Full transcript, all EWB, all analysis.
3. **`/dashboard`** — restructured around North Star Metric, Quick Actions, usage progress bars, recent calls, Next Best Action. Sidebar collapsible with 6 items.

**This phase is LAYOUT only.** Colors, glass effects, and visual polish (Glassmorphism refinement, shadows, animations) are a separate subsequent step — not in scope here.

Design reference: **Getclose.ai** (screenshot in user's external vault — researcher should fetch from web if needed). Key principle: clean, single-focus, information when needed, invisible otherwise.

</domain>

<decisions>
## Implementation Decisions

### Core UX Principle

- **D-01:** Kompaktmodus is the PRIMARY mode. The UX goal: Verkäufer schaut hin → sieht was er tun soll → schaut weg. 2 Sekunden. Not a secondary overlay — it's the default working surface during a call.
- **D-02:** Vollbild is optional — used before call start, after call ends, or when user wants detail. Not the default operational mode.
- **D-03:** **Layout only in this phase.** Colors, glass effects, Glassmorphism, shadows, and visual polish are deferred to a follow-up phase.

### Kompaktmodus — Structure (Document Picture-in-Picture)

- **D-04:** Kompaktmodus is implemented using the **Document Picture-in-Picture API** (`window.documentPictureInPicture`, Chrome 116+). NOT the legacy `video.requestPictureInPicture()` — no Canvas trick needed. The PiP window renders real HTML/CSS.
- **D-05:** PiP window must be **draggable and resizable** by the user (browser-native behavior via Document PiP API — the window is a real browser window). User positions and sizes it wherever they want on screen.
- **D-06 (nice-to-have):** Remember PiP window position and size between sessions via `localStorage`. Not blocking.

**Kompakt-Panel Layout:**
```
┌─ Mini-Header: [🔒 Cold Call]  00:42  [→ Vollbild] ────┐
├─ Tabs: [Opener] [Einwand] [Coaching] [EWB] ───────────┤
│                                                         │
│  [One Big KI Tip — large, readable]                    │
│  KI steuert automatisch, Tabs als Override             │
│                                                         │
├─ [EWB-1: "Zu teuer"]  [EWB-2: "Kein Budget"]  [Alle▾] ┤
└────────────────────────────────────────────────────────┘
```

- **D-07:** Four tabs at top: **[Opener] [Einwand] [Coaching] [EWB]** — always visible as manual override.
- **D-08:** Default: **KI steuers automatisch** welcher Tab aktiv ist und welcher Tipp angezeigt wird. Wenn KI etwas Neues erkennt → springt automatisch auf den relevantesten Tab/Inhalt, AUSSER der User hat einen Tab manuell fixiert.
- **D-09:** **Opener-Tab ist der erste Tab** — sichtbar und aktiv vor und beim Call-Start. Zeigt Einstiegssätze / Elevator Pitch aus dem aktiven Profil.
- **D-10:** **Opener-Erkennung ist inhaltsbasiert, nicht zeitbasiert.** KI analysiert Transkript und erkennt wann der Opener-Teil des Gesprächs abgeschlossen ist → schaltet dann automatisch auf Einwand/Coaching um. Falls KI den Opener nicht erkennt: User wechselt Tab manuell.
- **D-11:** EWB-Bereich zeigt **2 KI-ausgewählte EWB-Vorschläge** (wahrscheinlichste Einwände basierend auf Transkript-Kontext) + `"Alle ▾"` zum Aufklappen aller verfügbaren EWB. Nicht mehr alle 7 immer sichtbar.
- **D-12:** Mini-Header zeigt: Modus-Badge (Cold Call / Meeting) + Timer + Link zum Vollbild.

**Was NICHT im Kompakt erscheint (live):**
- **D-13:** Sprachanalyse (Redeanteil, Sprechtempo) → nur Post-Call
- **D-14:** Kaufbereitschaft als Prozentwert → nur Post-Call
- **D-15:** Phasen-Leiste → raus aus dem Live-View (bei Cold Call irrelevant, vermüllt UI)
- **D-16:** Volles Transkript → nur im Vollbild

### Vollbild (`/live` Fullscreen)

- **D-17:** Vollbild bleibt als vollständige Detailansicht: Transkript, alle EWB-Buttons, Sprachanalyse, Coaching, komplette AI-Cards.
- **D-18:** Chrome-Reduktion: Profile-Bar und Phasen-Bar werden aus dem always-visible Header-Stack entfernt oder collapsible — 4 Bars auf 1-2 reduzieren. Exakter Ansatz: Claude's Discretion.
- **D-19:** Vollbild bekommt ebenfalls den **Opener-Reiter** — als Section oder Tab in der AI-Panel-Spalte, nicht als eigene Chrome-Bar.

### Dashboard — Neues Layout

```
┌─ Sidebar (collapsible) ─┬─ Main ──────────────────────────────────────┐
│  [N] NERVE              │  ┌─ North Star ──┐  ┌─ Quick Actions ──────┐│
│  ─────                  │  │ Score Woche   │  │ [▶ Call starten]     ││
│  Dashboard              │  │ 72 / 100      │  │ [📚 Training]        ││
│  Live-Assistent         │  └───────────────┘  └──────────────────────┘│
│  Training               │  ┌─ Nutzung ─────────────────────────────── ┐│
│  Analytics              │  │ Live-Analyse  ████░░░░  420 / 2000 Min  ││
│  Profil                 │  │ Training      ██░░░░░░   12 / 50 Sessions││
│  Einstellungen          │  └──────────────────────────────────────────┘│
│  ─────────────          │  ┌─ Letzte Calls ────────────────────────────┐│
│  [👤] André · Pro       │  │ heute  Cold Call  Score: 78  12:34        ││
└─────────────────────────┘  │ gestern  Meeting  Score: 65  08:12        ││
                             │ ...                                        ││
                             └──────────────────────────────────────────┘│
                             ┌─ Next Best Action ────────────────────────┐│
                             │ 💡 Dein Opener-Score ist schwach           ││
                             │    → Opener-Training starten              ││
                             └──────────────────────────────────────────┘│
                             [Streak 🔥 5]  [Badge: Level 3]  (klein)    │
└────────────────────────────────────────────────────────────────────────┘
```

- **D-20:** **North Star Metric oben-links**: Score dieser Woche (Zahl + Trend).
- **D-21:** **Quick Actions oben-rechts**: Primär-CTA "Call starten" + "Training starten".
- **D-22:** **Nutzung als Fortschrittsbalken** (mitte): `Live-Analyse X/2000 Min` + `Training X/50 Sessions`. Ersetzt die 4 Statistik-Kreise.
- **D-23:** **Letzte 3-5 Calls**: Score, Dauer, Typ (Cold Call / Meeting). Kompakt, nicht ausladend.
- **D-24:** **Next Best Action** (unten): Kontextuelle Empfehlung basierend auf schwächstem Score-Bereich. Z.B. "Dein Opener-Score ist schwach — Opener-Training starten".
- **D-25:** Streak-Badge und Achievement-Badge: **klein und nicht dominant**. Sidebar-Fußzeile oder ein einzelner Chip unten auf der Seite.

**RAUS aus dem Dashboard:**
- **D-26:** KI-Motivationsspruch / Quote of the Day → gestrichen
- **D-27:** Statistik-Kreise (4-col SVG circles) → durch Fortschrittsbalken ersetzt
- **D-28:** Dominante Heatmap → gestrichen (oder auf Analytics verschoben)
- **D-29:** Große Achievements-Liste → auf kleinen Badge reduziert

### Sidebar

- **D-30:** Sidebar wird **collapsible** (Toggle-Button, Icon-only im eingeklappten Zustand).
- **D-31:** **6 Nav-Items**: Dashboard | Live-Assistent | Training | Analytics | Profil | Einstellungen. Alle anderen bestehenden Links (Coach, Methodik, Logs, Team, Changelog) werden ausgeblendet oder unter bestehende Items gefaltet.
- **D-32:** **User-Block unten-links**: User-Icon (Avatar initial) + Name + Plan-Badge (wie Claude UI). Bereits in `base.html` vorhanden — Layout anpassen, nicht neu bauen. Plan-Anzeige hinzufügen (wird aus `g.user` bzw. Org-Daten gelesen).
- **D-33:** "Analytics" entspricht der bestehenden `/logs`-Route (Logs/History). Kein neues Backend nötig — nur Umbenennung und Neupositionierung.

### Claude's Discretion

- Exakte Breite des Kompakt-PiP-Fensters (Standard-Startgröße)
- Wie "manuell fixiert" technisch implementiert wird (z.B. ein Lock-Icon im Tab, oder einfach dass ein manueller Tab-Klick den Auto-Modus pausiert bis KI neuen Inhalt liefert)
- Chrome-Reduction im Vollbild: wie Profile-Bar/Phasen-Bar verborgen wird (collapsible vs. merged vs. moved to sidebar)
- Next Best Action Logik: welcher Score-Bereich den Trigger auslöst

</decisions>

<specifics>
## Specific Ideas

- **2-Sekunden-Regel:** Every decision about what to show in Kompakt should be judged by: can the Verkäufer read and act on this in under 2 seconds while on a live call?
- **Opener-Erkennung:** Claude Haiku analysiert den Transkript-Beginn — wenn Opener-Keywords/-Patterns erkannt werden (Begrüßung, Gesprächseröffnung, Elevator Pitch fertig) → automatisch Tab wechseln. Threshold-Logik: Claude's Discretion.
- **KI wählt 2 EWBs:** Basierend auf letzten ~5 Transkript-Segmenten — welche Einwände am wahrscheinlichsten kommen? Ranking aus dem aktiven Profil `einwaende` Liste. Fallback: DACH-Standard-Liste aus Phase 4.2 D-11.
- **Sidebar wie Claude UI:** Unten-links User-Block mit Plan-Badge (z.B. "Pro" in teal). Inspiration: Claude.ai Sidebar-Footer.
- **Document PiP Fenster-Größe:** Startgröße ~380×280px. User kann frei anpassen. `localStorage` speichert letzte Größe/Position (nice-to-have).

</specifics>
