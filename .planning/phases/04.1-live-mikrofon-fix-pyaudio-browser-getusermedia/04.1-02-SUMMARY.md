- Mic lifecycle: starts on Socket.IO connect, stops on beenden() (before fetch) and on disconnect

## Decisions Made

- `workletNode.connect(audioCtx.destination)` required even though output is silent — without connecting to destination, some browsers garbage-collect the AudioWorklet node mid-session
- startMicStream() placed in socket.on('connect') handler — fires as soon as Socket.IO connection is up, before any transcript/coaching arrives
- stopMicStream() placed before the fetch('/api/beenden') call — ensures server-side Deepgram connection is closed via stop_live_session event before the beenden endpoint resets session state
- Client-side pause gate in onmessage (`!paused && micStarted`) is belt-and-suspenders; server also checks is_paused before processing audio_chunk
